# Airflow 노드/그룹 정체성 통합 + 접힘 시 경계 엣지 재배선

data 뷰에서 Airflow가 **두 개의 서로 다른 정체성**으로 모델링돼 있다: `node-airflow`(오케스트레이터 tool 노드 — data 엣지가 없어 orphan) 와 `node-airflow-group`(DAG 태스크 docling·presidio·kure를 감싸는 **합성 그룹 id**). 이 둘은 접기/펴기로 묶인 하나가 아니라 애초에 별개 노드라, "연결 없는 노드 표시"를 켜면 Airflow가 두 번 나타난다. 또한 그룹을 접으면 `buildNodesAndEdges`가 **내부 엣지(양 끝이 자식)만** 제거하고 **경계 엣지**(`s3-bronze → docling`, `kure → mock-api`)는 사라진 자식 id를 계속 가리켜 SvelteFlow가 drop → 접힌 카드가 파이프라인에서 완전히 단절되고 원점(-60,-60)에 겹쳐 렌더된다.

이전 계획서(`2026-07-21.airflow_그룹_접기펴기_config_모달.md`)는 "내부 엣지 숨김 + 자식 제외"만 명세했을 뿐 **정체성 통합**과 **경계 엣지 재배선**을 스펙에 넣지 않았다. 그래서 체크박스는 전부 통과했지만 결과 UX가 요구사항을 만족하지 못한다.

---

## 목표

- data 뷰에서 Airflow를 **단일 정체성**으로 표현한다 — 펼침 = 그룹 박스(내부 DAG 태스크), 접힘 = 단일 노드. 별도 orphan `node-airflow`가 그룹과 중복 표시되지 않는다.
- 그룹을 접어도 Airflow 단일 노드가 **데이터 흐름에 연결된 채로** 남는다: `s3-bronze → [Airflow] → mock-api` 유지.
- 접힌 Airflow 노드가 원점 겹침이 아니라 위상정렬 depth 기준 **흐름상 올바른 위치**에 배치된다.
- infra 뷰의 기존 동작(node-airflow가 오케스트레이터 노드로서 dependency 엣지 유지)은 회귀 없이 보존된다.

## 접근 방법

1. **정체성 통합 — 합성 그룹 id 제거**
   `node-airflow-group`을 없애고 DAG 태스크(docling·presidio·kure)의 `parentId`를 실제 `node-airflow`로 바꾼다(`topology.ts`, `realTopology.ts` 양쪽). 이후 data 뷰의 그룹 컨테이너 = 실제 `node-airflow` 노드. 이로써 "Airflow 하나" 요구를 데이터 모델 수준에서 충족한다.

2. **그룹 메타 파생화 — 하드코딩 제거**
   `buildNodesAndEdges`의 `groupMeta['node-airflow-group']` 하드코딩 맵을 제거하고, 그룹 노드 메타(displayName·icon·accent·role)를 **그룹 컨테이너 노드 자신**(`getToolEntry(node-airflow.tool)` 또는 노드 필드)에서 파생한다.

3. **data 뷰에서 컨테이너 노드의 leaf 렌더 억제**
   data 뷰에서 `node-airflow`는 그룹 박스/접힌 단일 노드로 렌더되므로, leaf tool 노드 배열에서는 제외한다(중복 렌더·orphan 방지). infra 뷰에서는 그룹핑을 하지 않으므로 기존대로 leaf로 렌더한다.

4. **접힘 시 경계 엣지 재배선 (핵심 신규 로직)**
   collapsed 그룹 G의 자식 집합 C에 대해 각 가시 엣지를 분기 처리한다:
   - 양 끝 ∈ C → drop (내부 엣지, 기존)
   - `from ∈ C, to ∉ C` → `(G, to)`로 재배선
   - `from ∉ C, to ∈ C` → `(from, G)`로 재배선
   - 둘 다 ∉ C → 유지
   재배선 결과 동일 `(source,target)` 쌍은 dedupe한다(여러 자식이 같은 외부 노드로 향할 때 중복 방지). 재배선은 **`computeDepths` 이전**에 수행해 접힌 G가 그래프 노드로서 depth 기반 흐름 위치를 얻도록 한다.

5. **접힌 그룹 노드의 위치 산출 변경**
   접힘 시 자식이 leaf 배열에서 제거돼 `absMinX`/`absMinY` 캡처가 불가능하므로, 접힌 그룹 노드 position을 재배선된 그래프의 `computeDepths` 결과에서 가져온다(원점 -60,-60 겹침 제거).

## 실행 시 필수 고려사항

① **infra 뷰 parentId 처리** — 자식 노드의 `parentId=node-airflow`가 infra 뷰(그룹 노드 미생성)에서 dangling parent를 만들지 않는지 확인한다. 현재도 `node-airflow-group` 참조로 동일 잠재 이슈가 있으므로, 그룹 미생성 뷰에서는 parentId/extent를 부여하지 않도록 분기한다.

② **재배선은 computeDepths 이전** — 접힌 그룹이 depth 그래프의 실제 노드로 참여해야 흐름 위치를 얻는다. 순서가 바뀌면 접힌 노드가 다시 원점에 떨어진다.

③ **엣지 dedupe & 라벨** — 여러 자식이 같은 외부 노드와 연결될 경우 재배선 후 중복 엣지가 생긴다. `(source,target)` 기준 dedupe하되, 서로 다른 viaTable 라벨이 충돌하면 대표 라벨 정책을 정한다(예: 첫 엣지 라벨 유지 또는 라벨 생략).

④ **node-airflow의 이중 역할** — data 뷰 = 그룹 컨테이너 / infra 뷰 = 오케스트레이터 leaf. 두 뷰에서 서로 다른 렌더 경로를 타므로 뷰 분기를 명확히 한다.

⑤ **config 모달 회귀** — 이전 계획서에서 구현한 그룹 제목 클릭 → config 모달(`onTitleClick`/`handleGroupSelect`)이 통합 후에도 `node-airflow` 기준으로 동작하는지 확인한다(groupId가 바뀌므로 `handleGroupSelect`의 자식 수집 로직 점검).

⑥ **접기 어포던스 가시성(별건, 범위 밖)** — expanded 헤더의 접기 버튼이 작은 `▾` 글리프뿐이라 발견성이 낮다. 본 계획 범위에는 넣지 않되, 필요 시 후속 별도 작업으로 분리한다.
