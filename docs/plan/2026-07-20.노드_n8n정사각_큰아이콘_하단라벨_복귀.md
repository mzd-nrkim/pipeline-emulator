# 노드 형태 복귀 — n8n식 정사각 큰 아이콘 + 카드 하단 라벨

현재 `ToolFlowNode`는 가로형 카드(180×64px, 좌 아이콘 + 우 제목·벤더 인라인)다. 이 형태가 실제 n8n 룩과 멀어 다시 **정사각 카드 + 중앙 큰 아이콘 + 카드 *바깥 아래* 라벨** 형태로 되돌린다. 이 형태는 직전 `done/2026-07-20.toolflownode_실제_n8n_룩_재디자인.md`가 만든 상태이며, 커밋 `4d5324c`(가로형 재구성) **직전**의 마크업·CSS와 동일하다. 즉 이 계획은 `4d5324c`의 **레이아웃 변경만 되돌리고**, 이후 커밋들이 얹은 기능(infra 핸들 숨김·deployStatus 배지·iconSpec 3-way 렌더)은 **보존**한다.

> **스코프 한정(사용자 명시)**: "다른 건 신경 안 써도 됨". 카테고리 색 분별(`e217aba`)·edge 조건 배지 등은 이번 작업의 개선/제거 대상이 아니다 — 정사각 카드에서 무해하게 잔존해도 되고, 필요 최소한만 정합한다. **핵심은 노드 모양(정사각+큰 아이콘+하단 라벨) 복귀 하나**다.

---

## 목표

- 노드가 **정사각(≈80px) 흰 카드 + 카드 중앙 큰 아이콘**으로 렌더된다(현재 가로 180×64px 폐기).
- `displayName`·`vendor`가 **카드 내부가 아니라 카드 바깥 아래** 중앙 정렬 블록으로 표시된다.
- 좌 target 1개 + 우 source(다중) **핸들 개수·세로 정렬이 회귀 없이** 유지되고, 핸들 `top` 공식이 정사각 카드 중심 기준으로 정합한다.
- **infra 뷰 핸들 `visibility:hidden`**(`isInfra` 조건)이 그대로 유지된다(기능 회귀 0).
- deployStatus 배지(예정/없음)·trigger ⚡·hover 상세 오버레이·nodeclick 드릴다운·outOfTeamScope 회색화가 새 레이아웃에서 회귀 없이 동작한다.
- `check`/`build`/`vitest`/`playwright`가 수치 근거와 함께 통과한다.

## 접근 방법

1. **레이아웃 되돌리기 (`ToolFlowNode.svelte`)** — 커밋 `4d5324c`의 레이아웃 부분을 역으로 적용한다:
   - 마크업: `.node-icon-area` + `.node-text-area`(카드 내부 2분할) → **아이콘을 카드 중앙 단일 배치**로 되돌리고, `.node-display-name`/`.node-vendor`를 **카드 형제(외부) `.node-label` 블록**으로 다시 분리한다.
   - CSS 치수: `--node-card-width:180px`·`--node-card-height:64px` → **`--node-card-size:80px`(정사각)** 로 복귀. `.node-card`를 `flex-direction:row` → 중앙 정렬 정사각으로. `.node-icon-area`/`.node-text-area`/좌측 구분선 스타일 제거, `.node-label`/`.node-display-name`/`.node-vendor`(카드 외부용) 스타일 재도입.
   - 루트 정렬: `.tool-flow-node { align-items: flex-start }` → `center`(카드+하단 라벨 중앙 정렬).
2. **핸들 공식 정합** — 핸들 `top`을 `var(--node-card-height)` → **`var(--node-card-size)`**(정사각 높이) 기준으로 되돌린다. target `top: calc(var(--node-card-size)/2)`, source 다중 `top: calc(var(--node-card-size) * (i+1)/(n+1))`. **라벨이 카드 밖 아래 형제이므로 핸들은 카드(상단 정사각) 중심에 정렬** — 루트 높이 증가가 핸들을 밀지 않도록 카드 높이 고정 변수만 참조.
3. **기능 보존(회귀 방지)** — 아래는 **건드리지 않는다**:
   - `isInfra ? 'visibility: hidden;'` 핸들 조건(`1cb25b6`) — 3개 Handle 모두 유지.
   - `resolvedDisplayName = displayName || label || 'Unnamed'` 폴백(PipelineGraphView minimal data 방어) — `label` 폴백 유지.
   - iconSpec 3-way 렌더(brand SVG / lucide / emoji), deployStatus PlannedBadge, trigger ⚡, hover `.node-detail`, outOfTeamScope 회색화, `selected` box-shadow.
   - 카테고리 색 클래스(`cat-*`/`--cat-*-border`)는 정사각 `.node-card` border에 무해 → **잔존 허용**(제거는 스코프 밖). 라벨 회색화는 외부 라벨 블록에 `node-label-muted` 재적용.
4. **두 뷰 자동 반영** — `ToolCanvasView`·`PipelineGraphView` 둘 다 `ToolFlowNode`를 공유하므로 카드 마크업 변경은 양쪽 자동 반영. PipelineGraphView는 `label`만 주입하므로 폴백 렌더('Unnamed'·vendor 없음)에서 빈 카드·크래시 없어야 한다.

## 실행 시 필수 고려사항

- **핸들 세로 정렬이 최대 회귀 지점**: 라벨을 카드 밖 아래로 빼면 컴포넌트 루트 높이가 커진다. 핸들은 xyflow 노드 루트 기준으로 배치되므로, `top`을 **정사각 카드 높이(=`--node-card-size`) 기준**으로 명시해 카드 중심(상단 정사각의 세로 중앙)에 맞춘다. `graph-lr-handle.spec.ts`(핸들 개수)로 실측. (이 형태는 `done/2026-07-20.toolflownode_실제_n8n_룩_재디자인.md`에서 이미 e2e 통과 이력 있음.)
- **infra 핸들 숨김 유지 필수**: `isInfra` 조건을 실수로 떨어뜨리면 infra 뷰에 핸들이 노출되는 회귀 — 3개 Handle 스타일 문자열에서 `isInfra ? 'visibility: hidden;'` 보존을 grep으로 확인.
- **하드코딩 색 금지**: `--surface`/`--border`/`--foreground`/`--muted-foreground`/`--primary`·`--node-accent`·oklch 토큰만. `#fff` 등 신규 하드코딩 0.
- **read-only 계약 불변**: `nodesDraggable` 등 편집 옵션·`+` 버튼 미도입.
- **nodeclick 드릴다운/pointer-events**: 외부 라벨 블록이 `.svelte-flow__node` 루트 클릭을 가로채지 않도록 pointer-events 주의(`node-detail-modal.spec.ts`).
