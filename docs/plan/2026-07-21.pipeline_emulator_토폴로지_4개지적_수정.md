# 파이프라인 에뮬레이터 토폴로지 4개 지적사항 수정

`/sample/pipeline` 캔버스에서 사용자가 제기한 4가지 문제를 수정한다. 핵심 원인은 (1) Airflow가 오케스트레이터 노드(`node-airflow`)와 합성 그룹 박스(`node-airflow-group`)로 이원화되어 서로 다른 뷰에서만 나타나는 구조, (2) `channels` 채널 지정 오류로 데이터 엣지가 인프라 뷰를 오염시키는 버그, (3) NiFi가 소스 DB에 의존함에도 엣지가 삭제된 모델링 불일치, (4) 그룹 라벨 하드코딩으로 인한 real 모드 config 모순이다. `/sample`은 `mockTopology`(topology.ts), `/real`은 `realTopology.ts`를 사용한다.

---

## 목표

- Airflow 오케스트레이터 노드와 DAG 그룹 박스를 **하나의 접기/펼치기 가능한 그룹**으로 통합해, 데이터 뷰에서 Airflow가 orphan(연결 없는 노드)으로 보이지 않게 된다.
- Airflow 그룹 멤버십이 dagId 보유 태스크 기준으로 **일관**되게 정리된다(gold_4_enrichment 편입).
- 그룹 ▶ 트리거 버튼과 그룹 노드 클릭이 **실제로 동작**한다.
- 인프라 뷰에 데이터 전용 노드(KURE·Mock API)·데이터 라벨(gold_chunked_documents)이 **새어 들어오지 않는다**.
- `node-nifi`의 소스 DB 의존이 토폴로지에 **반영**된다(mysql-container→nifi 복원).
- 그룹 라벨이 `node-airflow.config.executor` 값을 반영해 **real(SequentialExecutor)에서도 모순되지 않는다**.
- `mockTopology`/`realTopology` 드리프트가 문서화되어 이후 한쪽 수정 누락을 방지한다.

## 접근 방법

### A. Airflow 접기/펼치기 통합 (지적 #1)

1. **모델 통합**: `node-airflow`(오케스트레이터)와 합성 `node-airflow-group`을 개념적으로 하나로 본다. 그룹 박스 헤더가 곧 오케스트레이터다. `buildNodesAndEdges.ts`의 group 생성 로직(현 189-254행)에서 그룹 data에 `node-airflow`의 config(executor, dagIds/dagId, trigger)를 주입해, 그룹 헤더 클릭 시 `node-airflow` 상세를 열 수 있게 한다.
2. **접기/펼치기 상태**: `ToolCanvasView`(또는 pipeline `+page.svelte`)에 `airflowExpanded: boolean` 상태를 추가한다.
   - **펼침(기본)**: 현행처럼 그룹 박스 + 자식 DAG 태스크 노드 표시.
   - **접힘**: 자식 노드를 숨기고 그룹을 **단일 Airflow 노드**로 축소. 접힘 상태에서는 그룹 경계(s3-bronze→docling 진입, kure/mock-api→외부 진출) 엣지를 그룹 노드 하나로 리라우팅해 데이터 흐름이 끊기지 않게 한다.
   - 헤더에 접기/펼치기 토글(⌄/⌃) 추가.
3. **멤버십 일관화**: dagId를 가진 DAG 태스크만 그룹 자식으로 편입한다.
   - 편입: `node-docling`(silver_1)·`node-presidio`(silver_2)·`node-kure`(gold_3) — 현행 유지 + `node-mock-api`(gold_4_enrichment) **추가**(`parentId: node-airflow-group`).
   - **판단(es gold_5)**: `node-es`는 `outOfTeamScope:true`인 외부 Elasticsearch 싱크다. gold_5_field_mapping DAG가 ES에 쓰지만 ES 서비스 자체는 팀 외부 인프라이므로 **그룹 밖 싱크로 유지**한다(박스 안에 외부 스코프 노드를 넣지 않는다). 계획서 확정: es 미편입.
4. **박스 여백 정리**: `buildNodesAndEdges.ts`의 `PAD_BOTTOM=100`(208행)을 자식이 단일 행일 때 과다하므로 축소(예: `PAD_BOTTOM=40`) 또는 자식 행 수에 비례 계산으로 변경해 하단 빈 공간 제거.

### B. 인프라 뷰 오염 버그 (지적 #4A)

- `topology.ts:285` `node-kure → node-mock-api` 엣지의 `channels: ['data', 'dependency']`에서 `'dependency'`를 제거해 `['data']`로 수정한다. 이 엣지는 태스크 체인(데이터 흐름)이지 인프라 의존이 아니다. 수정 시 인프라 뷰에서 KURE·Mock API 노드와 `gold_chunked_documents` 부유 라벨이 사라진다. (`realTopology.ts`는 이미 `['data']`로 올바르므로 변경 불필요 — 확인만.)

### C. 그룹 트리거/클릭 死 수정 (지적 #4D)

- `AirflowGroupNode.svelte`의 ▶ 버튼은 `data.onTrigger` 부재 시 `dispatch('trigger')`하지만 소비처가 없다. 두 방법 중 택1:
  - (권장) `buildNodesAndEdges.ts` group data에 트리거 콜백을 연결할 수 있도록 하고, `ToolCanvasView`의 `onnodeclick`에서 `node.type === 'group'`일 때 `node-airflow`를 `onnodeselect`로 넘겨 상세/트리거를 열게 한다(현행 `if (node.type === 'group') return` 제거·대체).
  - 그룹 ▶ 버튼 클릭은 `onnodeselect(node-airflow)` 후 트리거 경로 재사용.

### D. real 그룹 라벨 하드코딩 수정 (지적 #2)

- `buildNodesAndEdges.ts:197` 하드코딩 `'Airflow (CeleryExecutor)'`를 `node-airflow.config.executor` 기반 동적 라벨(`Airflow (${executor})`)로 변경한다. topology에서 `node-airflow`를 찾아 executor를 읽는다.
- **검토**: `realTopology`는 `executor: 'SequentialExecutor'`인데 `valkey→airflow` (Celery 브로커) dependency 엣지가 존재한다 — SequentialExecutor는 Celery 브로커를 쓰지 않으므로 모순. real에서 이 엣지를 제거할지, 아니면 real executor를 실측 재확인할지는 작업 중 판단(실측 근거 우선). sample(CeleryExecutor)은 valkey→airflow 유지가 정상.

### E. 원본 DB → NiFi 엣지 복원 (지적 #3)

- `topology.ts`에서 삭제된 `node-mysql-container → node-nifi` 엣지를 복원한다. 근거: `node-nifi.config`에 `sqlQuery: 'SELECT * FROM docs'` + `connectionPool: 'dbcp2'`가 있어 JDBC로 소스 DB를 읽는다(mysql 의존). 기존 삭제 주석(topology.ts:59 "nifi는 mysql 미의존")은 config와 모순되므로 폐기하고 복원 근거로 교체.
- **검토(data 채널 승격)**: `mysql-container→debezium`·`mysql-container→nifi`를 `data` 채널로 승격하면 데이터 뷰에 소스 DB가 데이터 기원으로 표시된다. 다만 debezium CDC/NiFi가 소스 DB를 "읽는" 관계를 data로 볼지, 여전히 infra 의존으로 볼지는 뷰 일관성(데이터 뷰의 시작점 정의)에 영향 — 작업 중 두 뷰 스냅샷 비교로 결정. 기본안: dependency 유지 + nifi 엣지만 복원. 승격은 사용자 확인 후 선택 적용.

### F. real/sample 드리프트 문서화 (지적 #4E)

- `realTopology`에만 있는 `node-es-search`(gold_6), executor·index·database 차이 등을 두 파일 상단 주석에 상호 참조로 명기해, 한쪽 수정 시 다른 쪽 점검을 유도한다. (구조 통합은 범위 밖 — 인지·문서화만.)

## Verification

- **B**: 인프라 뷰(인프라 버튼)에서 KURE·Mock API 노드가 보이지 않고 `gold_chunked_documents` 라벨이 없다(스크린샷 대조). 데이터 뷰의 kure→mock-api 엣지·라벨은 유지.
- **D**: `/sample`은 그룹 헤더가 `Airflow (CeleryExecutor)`, `/real`은 `Airflow (SequentialExecutor)`로 config와 일치.
- **E**: 인프라 뷰에서 `MySQL 원본 DB → Apache NiFi` 엣지가 나타난다.
- **A**: 데이터 뷰에서 그룹 헤더 토글로 접힘(단일 Airflow 노드)↔펼침(DAG 박스) 전환되고, 접힘 상태에서도 s3-bronze→…→es/mysql 데이터 흐름이 끊기지 않는다. Mock API가 그룹 박스 안에 포함된다. 박스 하단 빈 여백이 사라진다. "연결 없는 노드 표시" 토글을 꺼도 Airflow가 데이터 뷰에 정상 표시된다(orphan 아님).
- **C**: 데이터 뷰에서 그룹 헤더/▶ 클릭 시 Airflow 노드 상세 모달이 열리고 트리거가 동작한다(dag_run_id 반환).
- **테스트/빌드**: `cd frontend && npm run test`(vitest) 통과, `npm run build` 성공, `npm run check`(svelte-check) 오류 없음.
- **회귀**: 데이터 뷰 fan-in(debezium/nifi/dam→s3-bronze)·fan-out(mock-api→es/mysql)·엣지 라벨(viaTable) 유지.
