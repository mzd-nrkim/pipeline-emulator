# Bug: MySQL 마이그레이션 트리거 생성 계정 권한 가이드 부재

`002_trigger_cdc.sql`의 `CREATE TRIGGER` 구문은 binlog 활성 환경에서 `SUPER` 권한이 필요하나, `emulator` 계정에 해당 권한이 없어 `ERROR 1419: You do not have the SUPER privilege and binary logging is enabled`가 발생했다. post-gate 실행 시 root 계정으로 재실행해야 했으며, migration 파일·README에 이 제약이 명시되어 있지 않아 재발 위험이 있다.

---

## 목표

- `db/migrations/` 내 DDL 파일에 필요 실행 계정을 주석으로 명시하여 post-gate 실행 가이드를 제공한다.
- `001_polling_index.sql`의 `CREATE INDEX IF NOT EXISTS` MySQL 미지원 문법을 제거하고 멱등 처리 가이드를 포함한다.
- 향후 마이그레이션 작성 시 `emulator` vs `root` 실행 기준 가이드가 `db/migrations/README.md`에 명시된다.

## 접근 방법

1. **`002_trigger_cdc.sql` 주석 추가**: 파일 상단에 `-- 실행 계정: root (SUPER 권한 필요 — CREATE TRIGGER + binlog_enabled)` 주석을 추가해 실행 시 혼선 방지
2. **`001_polling_index.sql` 문법 수정**: `CREATE INDEX IF NOT EXISTS`는 MySQL에서 지원하지 않음 → `CREATE INDEX {name}` + 중복 실행 시 `ERROR 1061` 무시 안내 주석으로 교체 (이미 현재 세션에서 수정됨 — 커밋 반영 여부 확인)
3. **`db/migrations/README.md` 신규 작성**: 마이그레이션 파일 실행 순서·계정별 실행 기준·멱등성 처리 방법을 문서화
