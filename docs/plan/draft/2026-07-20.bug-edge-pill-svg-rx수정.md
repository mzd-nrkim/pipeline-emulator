# edge 라벨 pill 배지 SVG rx/ry 미적용 수정

> 상태: 머지완료-통테대기

## 목표

Phase E(edge 조건 라벨 배지화)에서 `app.css`에 `rx`/`ry`를 CSS 속성으로 지정했으나 SVG 엘리먼트는 CSS `border-radius` 계열이 적용되지 않아 pill 형상이 실제로 렌더되지 않을 가능성이 있다. SvelteFlow 전용 속성(`labelBgBorderRadius`)으로 교체해 pill 모양을 보장한다.

## 실행 시 필수 고려사항

> plan-review 검토 중 실물 코드와 계획 전제가 상충하는 항목이 발견됐다. 구현 전 반드시 실물을 재확인한다.

- **① 전제 상충 (계획 premise ≠ 실물)**: `frontend/src/lib/canvas/buildNodesAndEdges.ts:194`에 이미 `labelBgBorderRadius: 9999`가 **설정돼 있다**. 계획서의 "`labelBgBorderRadius: 4` 추가" 전제는 현재 코드와 불일치(이미 9999). 또한 `frontend/src/app.css`의 `.svelte-flow__edge-textbg`(L159-163)에 `rx: 9999; ry: 9999;`가 **이미 존재**한다. 즉 TS prop·CSS 두 메커니즘이 모두 이미 적용된 상태 → 이 계획은 "미적용 추가"가 아니라 **"현재 pill 렌더 상태 실물 확인 → 실제 미렌더일 때만 원인 규명·수정"** 성격으로 재해석해야 한다.
- **주석 vs 구현 불일치**: `app.css:151` 주석은 "foreignObject 기반"이라 표기하나, `.svelte-flow__edge-textbg`에 `fill`/`rx`(SVG `<rect>` 전용 속성)를 쓰므로 실제 렌더는 `<rect>`로 추정된다. pill 미렌더 원인 조사 시 브라우저 DevTools로 실제 DOM(`rect` vs `foreignObject`)과 rx 적용값을 먼저 확인한다.
- **③ 동일 파일 편집 충돌**: `2026-07-20.bug-buildNodesEdges-groupIds-데드코드` 계획과 **동일 파일 `buildNodesAndEdges.ts`를 편집**한다(본 계획=엣지 라벨부 L188-198, 상대 계획=그룹 노드부 L135-138/L205-225). plan-run에서 **병렬 금지** — 직렬 배치하거나 한 에이전트로 묶어 실행한다.
- **② 패키지매니저**: 이 repo는 `package-lock.json`+`npm`(pipeline-emulator/CLAUDE.md는 `npm run check`/`npm run test:e2e` 사용). 계획서의 `pnpm`은 오기 — 게이트는 `npm` 명령을 쓴다.
