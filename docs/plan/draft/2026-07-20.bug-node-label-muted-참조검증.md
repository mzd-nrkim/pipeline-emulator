# ToolFlowNode .node-label-muted 삭제 클래스 참조 검증

> 상태: 미시작

## 목표

Phase N(ToolFlowNode 가로형 카드 재구성)에서 외부 라벨 `.node-label` 및 `.node-label-muted` CSS 클래스가 삭제됐으나, 다른 컴포넌트·스타일·테스트에서 해당 클래스를 참조하는지 검증하지 않은 상태로 머지됐다. 잔존 참조가 있으면 UI 스타일 깨짐이 발생할 수 있어 grep 확인 후 정리한다.

## 실행 시 필수 고려사항

> 검증 성격 작업 — grep 결과(잔존 참조 유무)에 따라 후속 편집 여부가 갈린다.

- **검증 후 분기 (실물 확인 후 결정)**: 이 계획은 `.node-label`/`.node-label-muted` 잔존 참조를 grep으로 확인하는 것이 1차 목표다. 잔존 참조 0건이면 **코드 변경 없이 종료**(삭제가 이미 안전했음을 확인)하고 상태를 갱신한다. 잔존 참조가 있을 때만 이전·제거·셀렉터 교체를 수행한다.
- **grep 오탐 주의**: `node-label`은 `node-label-muted`의 부분 문자열이자, 무관한 다른 클래스명(`node-label-xxx`)의 접두일 수 있다. grep 결과는 문맥(정의 vs 참조, CSS 셀렉터 vs 문자열)까지 확인한 뒤 판정한다.
- **② 패키지매니저**: 이 repo는 `package-lock.json`+`npm`. 계획서의 `pnpm run check`는 오기 — `cd frontend && npm run check` 사용.
- **동일 파일 충돌 없음**: 본 계획은 `frontend/src`(컴포넌트·스타일)·`frontend/e2e`를 다루며, 다른 두 계획(`buildNodesAndEdges.ts`)과 편집 파일이 겹치지 않아 **병렬 실행 가능**하다.

## 작업 목록

- [ ] A-1. src 잔존 참조 grep (path: frontend/src, 앵커: `grep -rn "node-label-muted\|node-label" frontend/src`, 의도: 삭제된 클래스의 컴포넌트·스타일 잔존 참조 탐지)

### B. 잔존 참조 정리 (A에서 참조 발견 시에만 수행)

- [ ] B-1. src 잔존 참조 정리 (path: 잔존 참조 발견 파일, 앵커: 해당 클래스 사용 지점, 의도: 카드 내부 스타일로 통합하거나 무효 참조 제거 — ToolFlowNode.svelte 새 DOM 구조 기준)
- [ ] B-2. e2e 셀렉터 교체 (path: 잔존 참조 발견 spec, 앵커: 삭제 클래스 셀렉터, 의도: ToolFlowNode.svelte 가로형 카드의 새 DOM 구조에 맞는 셀렉터로 교체)
