# F7. ES 다중 노드 (3-master) — 기능 계획서

> 작성일: 2026-07-14 / 상태: 대기 (트리거 발동 시 착수) / 우선순위: ★ (최하위)
> 방향전환 판단(2026-07-15): **보류**. F1 전제 + 데모 복잡도 대비 가치 불명확. HA 요구가 명시적으로 생길 때까지 연기.
> 인덱스: [pipeline-emulator-post-mvp-roadmap.md](./pipeline-emulator-post-mvp-roadmap.md)
> 토글: `ES=single|cluster` (기본 `single`)
> **전제: F1(검색 서빙 ES) `on`**

---

## 목표

ES 단일 노드를 3-master 클러스터로 확장해 HA·부하 분산을 시연한다.

## 전환 트리거

고가용성/부하가 시연 대상이 될 때. **단, F1 검색 서빙이 켜져 있어야 유효.**

## 작업

- [ ] ES 3-master 클러스터 구성 (`ES=cluster`)
- [ ] 부하 분산·장애 복구 시연 구성

## 검증 기준

- [ ] 노드 1개 다운 시 검색 지속 (HA)
- [ ] 부하 분산 확인
- [ ] `ES=single`로 복귀 가능

## 재사용 자산

- F1 검색 서빙 구성 (단일 노드 → 클러스터 확장)
- 원본 ES 3-master 클러스터 구성

## 리스크 & 헤지

| 리스크 | 헤지 |
|--------|------|
| 노트북 리소스 부담 큼 | **낮은 우선순위** — 데모 가치 대비 무거움. 실제 요구 발생 시에만 착수 |
| F1 미구현 시 무의미 | 의존성 명시 — F1 `on`이 아니면 토글 조작 불가(설정 메뉴 규칙) |

## 비고

전체 로드맵에서 우선순위 최하위. 노트북 환경에서 3-master는 리소스 부담이 크므로, 고가용성/부하가 명확한 시연 요구가 될 때만 착수한다.
