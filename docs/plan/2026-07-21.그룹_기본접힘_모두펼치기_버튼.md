# 그룹 기본 접힘 & 모두 펼치기/접기 버튼

> 상태: 미시작

현재 Airflow 그룹 노드는 캔버스 초기 진입 시 항상 **펼쳐진 상태**(자식 노드 노출)로 시작한다. 실제 파이프라인 개요를 파악하려면 그룹이 접혀 있어야 전체 흐름이 한눈에 들어오므로, 초기 로드 시 그룹을 접힌 상태로 렌더하고 한 번에 펼치는 버튼을 제공한다.

---

## 목표

- 데이터흐름 뷰 초기 진입 시 topology에 존재하는 모든 그룹 노드가 **접힌(collapsed) 상태**로 렌더된다.
- 캔버스 컨트롤 영역에 **"모두 펼치기 / 모두 접기"** 토글 버튼이 노출된다.
  - 모든 그룹이 접힌 상태 → "모두 펼치기" 버튼
  - 하나 이상 펼쳐진 그룹이 있을 때 → "모두 접기" 버튼
- 개별 접기/펼치기(기존 collapse-btn 클릭)는 동작이 유지된다.
- infra 뷰에서는 그룹 없음(기존 동작 보존).

## 접근 방법

1. **초기 collapsedGroups 계산 — ToolCanvasView.svelte**  
   `$effect`에서 topology가 처음 주입될 때, topology 내 모든 그룹 컨테이너 ID(자식을 가진 노드 ID)를 수집해 `collapsedGroups`의 초기값으로 세팅한다.  
   `topology`가 바뀔 때마다 재계산하되, 사용자가 이미 펼친 그룹은 보존(이미 `collapsedGroups`에 없는 것은 유지)하지 않고 새 topology에 맞게 리셋한다 — topology 교체는 뷰 전환 수준의 변화이므로 상태 리셋이 자연스럽다.  
   그룹 ID 계산: `topo.nodes.filter(n => topo.nodes.some(c => (c as any).parentId === n.id)).map(n => n.id)`

2. **"모두 펼치기 / 모두 접기" 버튼 — ToolCanvasView.svelte**  
   SvelteFlow `<Panel position="top-right">` (또는 기존 컨트롤 오버레이)에 버튼 배치.  
   - `allGroupIds` = topology에서 파생한 그룹 ID 전체 집합  
   - `allCollapsed = $derived([...allGroupIds].every(id => collapsedGroups.has(id)))`  
   - 버튼 클릭:  
     - allCollapsed → `collapsedGroups = new Set()` (모두 펼치기)  
     - !allCollapsed → `collapsedGroups = new Set(allGroupIds)` (모두 접기)  
   - 버튼 텍스트/아이콘: "▸ 모두 접기" / "▾ 모두 펼치기"  
   - data view에서만 표시 (`view === 'data'` 조건부).

3. **테스트 — buildNodesAndEdges 단위 테스트 회귀 확인**  
   `buildNodesAndEdges` 자체는 변경 없음. `ToolCanvasView`는 Svelte 컴포넌트라 단위테스트 불가 → e2e로 검증.

4. **e2e 테스트 추가 — airflow-group.spec.ts**  
   - "초기 진입 시 자식 노드(docling 등) DOM 미부착 확인" (collapsed 기본)  
   - "모두 펼치기 버튼 클릭 → 자식 노드 DOM 부착 확인"  
   - "모두 접기 버튼 클릭 → 자식 노드 DOM 미부착 확인"
