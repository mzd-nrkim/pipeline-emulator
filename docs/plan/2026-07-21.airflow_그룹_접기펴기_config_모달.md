# Airflow 그룹 노드 접기/펴기 + config 모달 트리거

> 상태: 통테통과-완료

DAG 캔버스에 Airflow 그룹 박스(box)가 도입됐으나 현재는 항상 펼쳐진 채로 고정된다. 자식 노드가 많을수록 박스가 넓어져 전체 토폴로지 가독성이 떨어진다. 또한 그룹 노드 헤더를 클릭해도 아무 반응이 없어 그룹 수준 정보를 확인하는 방법이 없다.

---

## 목표

- Airflow 그룹 노드를 클릭으로 접고 펼 수 있게 한다 (collapsed ↔ expanded 토글).
- 접힌 상태에서는 그룹 노드가 단일 컴팩트 노드처럼 렌더되며 자식 수를 표시한다.
- 그룹 헤더의 **제목(아이콘+이름) 클릭 시 config 모달을 열어** 그룹 메타데이터를 표시한다 (트리거 버튼 클릭은 제외).
- 접기/펴기 상태는 세션 내 유지된다 (페이지 새로고침 시 초기화 허용).

## 접근 방법

1. **collapsed 상태 관리 (ToolCanvasView)**  
   `collapsedGroups: Set<string>`를 Svelte `$state`로 관리한다. 토글 함수는 `groupId`를 받아 Set에 추가/제거하고, 이 값을 `buildNodesAndEdges` 및 `AirflowGroupNode`에 prop으로 전달한다.

2. **buildNodesAndEdges 확장**  
   `collapsedGroups` 인자를 추가한다. 그룹이 collapsed이면:
   - 해당 그룹의 자식 노드를 결과 배열에서 제외한다.
   - 그룹 노드의 `data.collapsed = true`, `data.childCount = N`을 설정한다.
   - 그룹 박스 크기를 컴팩트 단일 노드 크기(예: `width: 200, height: 100`)로 축소한다.
   - 그룹 내부를 지나는 엣지(그룹 자식 간)도 숨긴다 (엣지 `hidden` 플래그).

3. **AirflowGroupNode.svelte 분기 렌더**  
   `data.collapsed` prop에 따라 두 가지 뷰를 렌더한다:
   - **펼쳐진(expanded)**: 현재 점선 박스 + 헤더 레이아웃 유지.
   - **접힌(collapsed)**: 단일 카드 스타일(border-radius, 실선) + 아이콘·이름·`(N개)` 자식 수 표시 + 펼치기 chevron 아이콘.
   헤더의 트리거 버튼(`▶`)에는 `onclick:stopPropagation`을 유지하고, 제목 영역(아이콘+텍스트)에 별도 `ontitleclick` 이벤트를 발행한다.

4. **그룹 클릭 이벤트 분리 (ToolCanvasView)**  
   `onnodeclick`에서 `node.type === 'group'` 분기를 아래와 같이 세분화한다:
   - 이벤트 출처가 트리거 버튼 → 기존 trigger 처리 유지 (이미 버튼 자체에서 처리됨).
   - 이벤트 출처가 제목 영역 → `onGroupTitleClick(groupId)` 호출 → config 모달 표시.
   - 이벤트 출처가 collapse 토글 영역 → `toggleCollapse(groupId)` 호출.
   
   단, `@xyflow/svelte`의 `onnodeclick`은 클릭 원점(target)을 직접 주지 않으므로, **컴포넌트 내부에서 커스텀 이벤트**(`dispatch('titleclick')`, `dispatch('togglecollapse')`)를 발행하고, `ToolCanvasView`가 바인딩하는 방식으로 구현한다.

5. **config 모달 — 그룹 뷰**  
   `+page.svelte`의 `handleNodeSelect`를 확장해 `ToolNode | GroupMeta | null`을 수용하거나, 별도 `selectedGroup` 상태를 추가한다. 모달 내 그룹 탭에서는 그룹 메타(displayName, role, icon, accent, childCount, 자식 노드 ID 목록)를 읽기 전용으로 표시한다 (설정 편집 필드는 필요 시 추후 추가).

## 실행 시 필수 고려사항

① **@xyflow/svelte 커스텀 이벤트 버블링 불가 — data 콜백 방식 필수**  
계획서 접근 방법 4에서 "커스텀 이벤트 `dispatch('titleclick')`, `dispatch('togglecollapse')`를 발행하고 ToolCanvasView가 바인딩"이라고 했으나, `@xyflow/svelte`가 내부에서 렌더하는 노드 컴포넌트의 Svelte 이벤트는 SvelteFlow 경계를 넘어 상위 ToolCanvasView로 버블링되지 않는다. 기존 트리거 버튼도 동일 이유로 `data.onTrigger?.()` 콜백 방식을 쓴다. 따라서 `dispatch` 대신 **`data.onTitleClick?.()` / `data.onToggleCollapse?.()` 콜백 props** 방식을 써야 한다. ToolCanvasView의 `$effect`에서 group 노드 data에 클로저 콜백을 후처리로 주입한다.

② **FlowNode.data 인터페이스 확장 필요 (A-1에서 처리)**  
현재 `buildNodesAndEdges.ts:9-32`의 `FlowNode.data`에는 `collapsed`, `childCount`, `onTitleClick`, `onToggleCollapse` 필드가 없다. A-1에서 함수 시그니처 확장과 함께 인터페이스 필드를 추가한다.

③ **`GroupMeta` 타입 신규 정의 필요 (D-1에서 처리)**  
`types.ts`에 `GroupMeta` 인터페이스가 없다. D-1에서 `types.ts`에 추가하거나 +page.svelte 인라인 타입으로 정의한다.

④ **Phase 의존성 — A·B 병렬, C는 A+B 완료 후**  
A(`buildNodesAndEdges`)와 B(`AirflowGroupNode.svelte`)는 상호 독립적으로 수정 가능하나, C(`ToolCanvasView`)는 A의 새 인터페이스와 B의 새 props를 모두 소비하므로 A·B 완료 후 실행한다. D는 C 완료 후 실행.

⑤ **pointer-events 변경 범위**  
현재 `.airflow-group-header`가 `pointer-events: none`(AirflowGroupNode.svelte:79). 제목 영역과 chevron 버튼에만 `pointer-events: all`을 추가한다. 그룹 박스 전체(`pointer-events: none`)는 유지 — 자식 노드 클릭과의 충돌 방지를 위해.
