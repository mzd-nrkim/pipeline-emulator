# 인프라 뷰 엣지 노드 우회 — Plan A: 런타임 장애물 회피 커스텀 엣지

> 상태: 대기중 (Plan B 실패 시 전환)
> 선행 계획: `2026-07-21.인프라뷰_LR계층_꺾은선엣지_재설계.md` (머지완료)

## 문제

Plan B(빌드타임 pre-computed path)의 한계:
- 핸들 위치를 `pos.x + 80`으로 고정 추정 → 실제 측정 폭(measured.width)과 다를 때 경로와 핸들이 어긋난다.
- borderRadius 라운딩을 `getSmoothStepPath` 없이 직접 구현해야 한다.

Plan A는 xyflow 스토어에서 **측정된 실제 노드 크기·위치**를 읽어 런타임에 장애물 회피 경로를 계산한다.

## 목표

- 모든 인프라 엣지가 어떤 노드도 관통하지 않는다.
- 경로가 실제 측정된 핸들 위치에 정확히 앵커된다.
- 토폴로지·노드 크기가 바뀌어도 자동으로 재라우팅된다.
- vitest 전량 통과, e2e 스크린샷 노드 관통 없음.

## 접근 방법

### 1. InfraOrthogonalEdge.svelte

xyflow의 `EdgeProps`를 받는 커스텀 엣지 컴포넌트. 렌더 시점에 xyflow 스토어에서 노드 목록을 읽어 장애물 회피 경로를 직접 계산한다.

#### 런타임 라우팅 알고리즘

```
입력:
  hx1, hy1  — source right 핸들 좌표 (EdgeProps.sourceX/Y)
  hx2, hy2  — target left 핸들 좌표 (EdgeProps.targetX/Y)
  nodes     — 스토어에서 읽은 FlowNode[] (현재 뷰)

계산:
  1. 중간 노드 탐색:
     midNodes = nodes where (node.position.x + OFF < hx2 - OFF)
                           AND (node.position.x > hx1 + OFF)
  2. 우회 y 계산:
     If midNodes is empty:
       → getSmoothStepPath 사용 (일반 smoothstep 위임)
     Else:
       minY = min(midNode.position.y for midNode in midNodes)
       ROUTE_Y = minY - MARGIN      (노드 위 MARGIN px)
  3. 6-point 경로 생성 (Plan B와 동일 공식, 단 hx·hy가 정확한 측정값):
     M hx1 hy1
     L (hx1+OFF) hy1
     L (hx1+OFF) ROUTE_Y
     L (hx2-OFF) ROUTE_Y
     L (hx2-OFF) hy2
     L hx2 hy2
  4. 코너 라운딩: borderRadius=6 적용 (getBend 동등 함수)
```

#### xyflow 스토어 접근

```svelte
<script lang="ts">
  import { useNodes, type EdgeProps } from '@xyflow/svelte';
  const nodes = useNodes();
  // ... routing 계산
</script>
```

`useNodes()`는 현재 xyflow 인스턴스의 노드 배열을 반환하는 Svelte 스토어.
노드 위치·크기가 바뀌면 자동 재계산.

### 2. buildNodesAndEdges.ts 변경

- `data` 에 별도 경로 정보를 넣지 않는다 — 런타임 계산이므로.

## Plan A vs Plan B 결정 매트릭스

| 기준 | Plan B (빌드타임) | Plan A (런타임) |
|------|-----------------|----------------|
| 구현 복잡도 | 낮음 (svgPath 문자열 생성) | 높음 (store 구독·조건 분기) |
| 핸들 정밀도 | 추정값 (±0~10px) | 정확 (EdgeProps.sourceX/Y) |
| 노드 크기 변경 반응 | 없음 (재빌드 시 반영) | 즉시 반응 |
| 라운딩 구현 | 직접 구현 필요 | `getSmoothStepPath` 위임 가능 |
| e2e 신뢰성 | 경로 고정 → 단언 쉬움 | 노드 크기 측정 완료 후만 신뢰 |

**권장**: Plan B 먼저 시도. 핸들 추정 오차가 시각적으로 무시 가능하면 B로 완료.
오차가 눈에 띄거나 미관이 불만족스러우면 Plan A로 전환.
