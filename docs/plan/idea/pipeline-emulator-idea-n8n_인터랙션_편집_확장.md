# [IDEA] n8n 스타일 인터랙션·레이아웃·편집 확장

n8n 개선 제안 7항 중 **편집기 성격을 띠는 항목**(자유형식 레이아웃·계층적 노드 드릴다운·`+`버튼 인터랙티브 편집·편집 사이드바)을 모은 **아이디어 단계** 문서다. PipeScale은 현재 **read-only 파이프라인 시각화/에뮬레이터**이므로, 이 항목들은 "시각화 도구 → (부분) 편집 가능 도구"로의 **성격 전환**을 수반한다. 착수 전 **제품 방향 결정**이 필요해 활성 계획(`docs/plan/`)이 아닌 `idea/`에 둔다. 시각화 개선(1·2·3·6)은 별도 활성 계획 `docs/plan/2026-07-20.n8n_시각화_스타일_개선_노드형태_edge라벨_색상.md`로 진행 중.

> **현황 실측** — "상세 정보 패널"은 부분적으로 **이미 존재**한다: `node-detail-modal`(Playwright `node-detail-modal.spec.ts`)로 노드 클릭 시 상세·실행 이력 모달이 뜬다. 따라서 제안 7(사이드바)·4(계층 드릴다운)의 *read-only 부분*은 신규가 아니라 **기존 모달의 확장**으로 재해석해야 한다.

---

## 목표 (전환 시 달성 상태 — 방향 확정 후 활성화)

- **자유형식 레이아웃**: 노드를 캔버스 임의 위치에 드래그 배치하고 위치가 저장/복원된다(현재는 force/고정 레이아웃 자동 배치, 드래그 비활성).
- **계층적 노드 드릴다운**: n8n AI Agent처럼 특정 노드 선택 시 하위 구성요소(예: DAG task, PII layer1/2, enrich 하위 스텝)를 컨텍스트로 펼쳐 본다 — 기존 모달을 **인라인/사이드 확장**으로 승격.
- **인터랙티브 액션**: 노드 hover 시 액션 버튼(`+`·상세·재실행 등)을 노출 — 단, read-only 범위에서 "재실행/조회" 위주인지 "노드 추가/편집"까지인지 결정.
- **좌우 패널**: 좌(카탈로그/템플릿)·우(선택 노드 상세·변수) 패널로 화면 정보 밀도를 높인다 — 우패널은 기존 모달을 대체/보완.

## 착수 전 결정 필요 (D — 제품 방향)

1. **PipeScale의 정체성**: read-only 시각화 유지 vs (부분) 편집 도구 전환. → 4·5·7의 범위를 좌우하는 최상위 결정.
2. **자유배치 vs 자동 레이아웃**: 자유 드래그 도입 시 기존 force/고정 레이아웃(`forceLayout.ts`)과의 관계 — 자동 초기배치 후 수동 미세조정 허용? 위치 영속화 저장소(localStorage vs 백엔드)?
3. **`+`버튼의 의미**: 워크플로우에 노드를 *추가*(편집)인지, 인접 노드로 *네비게이트/드릴다운*(조회)인지. 후자면 read-only 유지 가능.
4. **사이드 패널 vs 모달**: 기존 `node-detail-modal`을 사이드 패널로 이전할지, 병존할지. e2e(`node-detail-modal.spec.ts`) 회귀 범위 결정.

## 접근 방법 (스케치 — 결정 후 상세화)

1. **드래그 활성화**: SvelteFlow `nodesDraggable`/`onnodedragstop`으로 위치 캡처 → 영속화. 초기 배치는 기존 force/고정 유지, 사용자 이동분만 오버라이드 저장.
2. **드릴다운**: 노드별 하위 구성요소 데이터 모델 필요(현재 `toolCatalog`는 단일 노드 단위). DAG task/PII layer/enrich 스텝을 하위 노드로 표현하는 **중첩 그래프** 또는 **모달 내 서브그래프**.
3. **hover 액션**: `ToolFlowNode`에 hover 시 절대배치 액션 바. read-only면 "상세·실행이력·재실행 트리거"로 한정.
4. **패널 레이아웃**: 라우트 레이아웃(`[mode=mode]/pipeline/+page.svelte`)에 좌우 그리드 슬롯 추가, 우패널은 선택 노드 스토어 구독.
