# DAG 뷰 시각 개선 — 수정보고서

- **작업일**: 2026-07-20
- **범위**: 프론트엔드 순수 시각(CSS/마크업) 개선. 스키마·API·DB 영향 없음.
- **상태**: 완료 (main 머지됨). 커밋 `0f85b06`, `008b7b8`.

## 배경 / 요청

파이프라인 DAG 뷰에 대한 3가지 시각 개선 요청:

1. DAG 노드 글자가 너무 작음(기본 폰트조차 작음).
2. `readonly`가 빨간색으로 표시됨 — grayout이 아니라 빨강이라 의미가 어긋남.
3. 기타 시각 개선 제안.

## 문제 진단

| # | 증상 | 원인 |
|---|------|------|
| 1 | DAG 노드 라벨이 작음 | `ToolFlowNode` 라벨 폰트가 `0.75rem`(이름)/`0.65rem`(vendor)/`0.6rem`(상세). @xyflow `fitView`가 노드 수 많을 때 캔버스를 축소해 더 작아짐 |
| 2 | readonly가 빨강 | config 패널 배지 `bg-red-100 text-red-700`(🔴), `ToolFlowNode` hover 배지 🔴. 빨강은 "에러/위험" 시맨틱 → "변경불가(비활성)"에 부적절 |
| 3 | 전반 가독성 | config 패널 폰트 9~10px, 엣지 condition 라벨 무스타일, 색배지 다크모드 미대응 |

## 적용 내용

### #1 DAG 라벨 폰트 확대 — `ToolFlowNode.svelte`

- `.node-display-name` `0.75rem → 0.9rem`
- `.node-vendor` `0.65rem → 0.78rem`
- `.detail-row` `0.6rem → 0.72rem`
- `.node-label` `max-width` `+24px → +48px` (긴 이름 잘림 완화)

### #2 readonly 빨강 → 중립 회색(잠김)

- config 배지: `bg-red-100 text-red-700 border-red-300` 🔴 → `bg-surface-muted text-muted-foreground border-border` 🔒
- `ToolFlowNode` hover 배지: `applyModeConfig.readonly` 이모지 🔴 → 🔒
- 근거: readonly는 위험이 아니라 disabled 시맨틱. 인풋은 이미 `disabled:opacity-50`로 grayout 되어 있어 배지 색만 어긋나 있었음.

### #3 추가 개선

- **config 패널 폰트 확대**: 라벨·인풋·배지 9~10px → 10~11px (`pipeline/+page.svelte` 설정 폼)
- **엣지 라벨 스타일링**: `.svelte-flow__edge-text` 11px·`.svelte-flow__edge-textbg` 배경 지정 — 두 그래프 뷰(`ToolCanvasView`, `PipelineGraphView`)
- **fitView 과축소 방지**: `fitViewOptions={{ minZoom: 0.6, maxZoom: 1.2 }}` — 노드 많을 때 글자 작아짐 완화
- **배지 다크모드 대응**: runtime/restart/code 배지에 `dark:` variant 보정, readonly는 디자인 토큰(`surface-muted`/`muted-foreground`)으로 통일
