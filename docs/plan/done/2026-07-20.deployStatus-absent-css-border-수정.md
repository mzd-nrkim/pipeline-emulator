# deployStatus absent CSS border-style wrapper 적용 오류 수정

> 상태: 통테통과-완료

## 목표

`ToolFlowNode.svelte`의 `.status-absent { border-style: dashed }` 규칙이 wrapper 요소(`.tool-flow-node`)에 적용돼 실제 카드(`.node-card`)에 점선 테두리가 보이지 않는 버그를 수정한다.

`deployStatus: 'absent'` 노드 카드에 점선 테두리가 시각적으로 표시되어야 한다.

## 배경

Phase S 에이전트(agent-ac77952a9c8ffef33)가 `.status-absent { border-style: dashed }`를 wrapper 셀렉터에 추가했으나, 실제 테두리는 `.node-card` 내부 요소에 있다. 기존 `.out-of-scope .node-card { border-left: 4px dashed ... }` 패턴과 달리 자식 셀렉터를 쓰지 않아 시각적으로 동작하지 않을 가능성이 있다.

실제 동작 여부를 `deployStatus='absent'` 지정 노드로 브라우저에서 확인 후 결함이 있으면 자식 셀렉터 패턴으로 수정한다.

## 접근 방법

1. `topology.ts`에서 임시로 한 노드에 `deployStatus: 'absent'` 지정해 브라우저에서 점선 테두리 확인.
2. 테두리 미표시 시 `ToolFlowNode.svelte`의 `.status-absent` CSS를 `.status-absent .node-card { border-style: dashed }` 또는 `.status-absent .node-card { border: 1px dashed currentColor }` 자식 셀렉터로 수정.
3. `deployStatus: 'planned'`(.status-planned)도 동일 패턴인지 교차 확인.
4. 임시 topology 변경은 확인 후 원복 (커밋 금지).

## 실행 시 필수 고려사항

- **결함은 실측 검증됨(구조적 확실)**: `.status-absent`(ToolFlowNode.svelte:298-301)는 wrapper `.tool-flow-node`에 `border-style: dashed`만 지정하는데 wrapper엔 `border-width`가 없다(테두리는 `.node-card`:172 `border:1px solid`에만 존재). `border-style`만 있고 `border-width`가 0인 요소는 테두리가 렌더되지 않으므로 점선은 **보이지 않는다**. `.out-of-scope .node-card`(:303)가 이미 자식 셀렉터 패턴을 쓰는 것과 대조된다 → 수정 방향은 `.status-absent .node-card { border-style: dashed }`로 확정적.
- **`.status-planned`은 동일 결함 아님**: `.status-planned`(:294-296)는 `opacity`만 지정하고 border를 건드리지 않는다 → 렌더 결함 없음. 점선을 **요구하는 게 아니라면** 수정 불필요. (점선 요구 시에만 `.status-planned .node-card` 자식 셀렉터 추가 — 실물 확인 후 결정)
- **임시 데이터 원복과 e2e 회귀 스펙의 긴장(중요)**: 이 계획은 검증용 `deployStatus:'absent'` 데이터를 확인 후 원복한다(커밋 금지). 그런데 topology.ts에 **영구적으로 `absent`인 노드가 하나도 없다**(grep 0건). 따라서 원복 이후(=Z-post 시점)에는 absent 카드를 렌더하는 노드가 없어 **자동 e2e 회귀 스펙이 대상 데이터를 못 찾는다**. 두 갈래(실물 확인 후 결정):
  - (A·권장) **영구 absent 픽스처 커밋** — `node-kibana`는 topology 주석(:39)에서 이미 "없음"(=absent)으로 문서화돼 있으므로, `node-kibana`에 `deployStatus:'absent'`를 **영구 반영**하면 의미도 정합하고 e2e 회귀 대상이 생긴다. 이 경우 `deployStatus-border.spec.ts` 신규 작성(Z-post). 단 infra 노드라 핸들 숨김·배지 상호작용을 함께 확인해야 한다(스코프 소폭 확대).
  - (B) **A-1 1회 수동/playwright 확인만** — 임시 데이터 상태에서 `.node-card` computed `border-style=dashed`를 1회 실측하고 원복. 영구 회귀 스펙 없음(다음 회귀는 잡지 못함). 이 경우 Z-post e2e 항목은 제거된다.
- **하드코딩 색 금지**: 기존 토큰(`--border` 등)만 사용. 신규 hex 0.

**Right-BICEP**
