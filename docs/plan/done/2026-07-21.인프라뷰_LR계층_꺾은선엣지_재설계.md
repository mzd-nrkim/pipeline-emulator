# 인프라 뷰 LR 계층 배치·꺾은선 엣지 재설계

> 상태: 머지완료 (vitest 87 pass · svelte-check 0 errors · playwright 시각검증 통과)
> 비고: 계획서 없이 구현·머지가 선행된 뒤 소급 작성된 계획서. 워크플로 규약(계획서 선작성) 위반을 기록으로 남긴다. 커밋 990040b, 머지 8812d00.

`/sample/pipeline` 인프라 뷰에 두 가지 문제가 있었다.

1. **가독성 (직선 스파게티)**: 인프라 뷰가 force-directed 배치(`forceLayout.ts`, d3-force 원형 blob)로 노드를 흩뿌리고, 그 사이를 중심-대-중심 **직선 점선**(`InfraFloatingEdge.svelte`의 `M p1 L p2`)으로 이었다. 방향성(위상)이 없고 교차가 많아 인프라 의존 그래프로서 가독성이 낮았다.
2. **노드-엣지 연결 끊김 (렌더 버그)**: `InfraFloatingEdge.svelte`가 `srcNode.width ?? 180` / `srcNode.height ?? 64`로 bbox 교점을 계산했는데, @xyflow/svelte v1은 측정된 실제 크기를 `node.measured.width/height`에 저장하고 `node.width/height`는 사용자가 명시하지 않으면 `undefined`다. 이 노드들은 크기를 지정하지 않으므로 **항상 180×64 유령 박스**로 경계점을 찍었다. 실제 노드 카드는 `ToolFlowNode.svelte`의 `--node-card-size: 80px` 기준 80×80이라, 엣지 끝점이 실제 노드보다 가로로 ~50px 바깥에 생겨 **점선이 노드에 닿지 않고 허공에 떴다.** 인프라 뷰에서 Handle이 `visibility:hidden`이라 접점 표시조차 없어 더 떠 보였다.

---

## 목표

- 인프라 뷰 노드가 의존 방향 기반 **LR 계층 배치**(depth = X축 좌→우, 같은 depth = Y축 분산)로 렌더된다 — force blob 제거.
- 인프라 뷰 엣지가 **꺾은선(직교, smoothstep)**으로 라우팅되고, 실제 노드 Handle 경계에 **정확히 앵커**된다 — 연결 끊김 버그 해소.
- 인프라 뷰 Handle이 표시되어 직교 엣지의 접점이 보인다.
- 유령 박스 계산 로직·force 배치 등 관련 dead code가 제거된다.
- 단위테스트가 새 엣지 타입(smoothstep)을 검증하며 전량 통과한다.

## 접근 방법

1. **배치 통합** — `buildNodesAndEdges.ts`: `view === 'infra'` 분기의 `computeForceLayout` 호출을 제거하고, data 뷰가 이미 쓰던 위상정렬 depth 배치(`computeDepths` + `x = depth * COL_GAP`, `y = idx * ROW_GAP`)를 data·infra 공통으로 사용한다. `computeForceLayout` import 제거.
2. **엣지 타입 교체** — `buildNodesAndEdges.ts`: 인프라 엣지의 `type: 'infra-floating'`(커스텀 직선)을 내장 `type: 'smoothstep'`(직교)으로 교체. 내장 edge는 실제 Handle 좌표에 앵커되므로 유령 박스 계산이 불필요해진다.
3. **edge 등록 제거** — `ToolCanvasView.svelte`: `edgeTypes={{ 'infra-floating': InfraFloatingEdge }}` 등록과 `InfraFloatingEdge` import 제거(smoothstep은 내장이라 등록 불필요).
4. **Handle 표시** — `ToolFlowNode.svelte`: target/source Handle style의 `{isInfra ? 'visibility: hidden;' : ''}` 제거(3곳). 미사용이 된 `isInfra` 파생값 제거.
5. **dead code 삭제** — `forceLayout.ts`, `floatingEdge.ts`, `floatingEdge.test.ts`, `InfraFloatingEdge.svelte` 삭제.
6. **테스트 갱신** — `buildNodesAndEdges.test.ts`: 인프라 엣지 `type` 단언을 `'infra-floating'` → `'smoothstep'`으로 변경. 인프라 좌표 유한성·결정성 테스트는 계층 배치에서도 유효하므로 유지.

## 실행 시 필수 고려사항

- **xyflow v1 측정 좌표 API**: 이 버그의 근본 원인이 `node.width`(미측정 undefined) vs `node.measured.width`(측정값) 혼동이다. 앞으로 floating/custom edge에서 노드 실측 크기가 필요하면 `node.measured`를 참조해야 한다. 이번 수정은 내장 smoothstep으로 전환해 이 API 의존 자체를 제거했다.
- **LR 방향 정합성**: 계층 배치에서 source가 낮은 depth(좌), target이 높은 depth(우)에 놓이므로 Right(source)/Left(target) Handle 정렬과 일치한다 — smoothstep이 좌→우로 깔끔하게 꺾인다. 인프라 엣지에 condition이 없어 기본 Right source handle을 쓴다.
- **data 뷰 무영향**: 배치 로직 통합은 data 뷰가 원래 쓰던 코드 경로라 data 뷰 동작·그룹 노드 후처리(data 전용)에 변화가 없어야 한다. 테스트로 회귀 확인.
