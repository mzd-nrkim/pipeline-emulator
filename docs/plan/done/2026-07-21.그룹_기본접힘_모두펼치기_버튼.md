# 그룹 기본 접힘 & 모두 펼치기/접기 버튼

> 상태: 통테통과-완료

현재 Airflow 그룹 노드는 캔버스 초기 진입 시 항상 **펼쳐진 상태**(자식 노드 노출)로 시작한다. 실제 파이프라인 개요를 파악하려면 그룹이 접혀 있어야 전체 흐름이 한눈에 들어오므로, 초기 로드 시 그룹을 접힌 상태로 렌더하고 한 번에 펼치는 버튼을 제공한다.

---

## 목표

- 데이터흐름 뷰 초기 진입 시 topology에 존재하는 모든 그룹 노드가 **접힌(collapsed) 상태**로 렌더된다.
- 캔버스 컨트롤 영역에 **"모두 펼치기 / 모두 접기"** 토글 버튼이 노출된다.
  - 모든 그룹이 접힌 상태 → "모두 펼치기" 버튼
  - 하나 이상 펼쳐진 그룹이 있을 때 → "모두 접기" 버튼
- 개별 접기/펼치기(기존 collapse-btn 클릭)는 동작이 유지된다.
- infra 뷰에서는 그룹 없음(기존 동작 보존).

## 접근 방법

1. **초기 collapsedGroups 계산 — ToolCanvasView.svelte**  
   `$effect`에서 topology가 처음 주입될 때, topology 내 모든 그룹 컨테이너 ID(자식을 가진 노드 ID)를 수집해 `collapsedGroups`의 초기값으로 세팅한다.  
   `topology`가 바뀔 때마다 재계산하되, 사용자가 이미 펼친 그룹은 보존(이미 `collapsedGroups`에 없는 것은 유지)하지 않고 새 topology에 맞게 리셋한다 — topology 교체는 뷰 전환 수준의 변화이므로 상태 리셋이 자연스럽다.  
   그룹 ID 계산: `topo.nodes.filter(n => topo.nodes.some(c => (c as any).parentId === n.id)).map(n => n.id)`

2. **"모두 펼치기 / 모두 접기" 버튼 — ToolCanvasView.svelte**  
   SvelteFlow `<Panel position="top-right">` (또는 기존 컨트롤 오버레이)에 버튼 배치.  
   - `allGroupIds` = topology에서 파생한 그룹 ID 전체 집합  
   - `allCollapsed = $derived([...allGroupIds].every(id => collapsedGroups.has(id)))`  
   - 버튼 클릭:  
     - allCollapsed → `collapsedGroups = new Set()` (모두 펼치기)  
     - !allCollapsed → `collapsedGroups = new Set(allGroupIds)` (모두 접기)  
   - 버튼 텍스트/아이콘: "▸ 모두 접기" / "▾ 모두 펼치기"  
   - data view에서만 표시 (`view === 'data'` 조건부).

3. **테스트 — buildNodesAndEdges 단위 테스트 회귀 확인**  
   `buildNodesAndEdges` 자체는 변경 없음. `ToolCanvasView`는 Svelte 컴포넌트라 단위테스트 불가 → e2e로 검증.

4. **e2e 테스트 추가 — airflow-group.spec.ts**  
   - "초기 진입 시 자식 노드(docling 등) DOM 미부착 확인" (collapsed 기본)  
   - "모두 펼치기 버튼 클릭 → 자식 노드 DOM 부착 확인"  
   - "모두 접기 버튼 클릭 → 자식 노드 DOM 미부착 확인"

## 실행 시 필수 고려사항

> plan-review 검토 중 발견 사항 — plan-run 실행 전 반드시 숙지.

**① 회귀 이유·범위 (높음 — 기존 e2e 다수가 "초기 펼침" 전제)**
- `frontend/e2e/airflow-group.spec.ts`(기존, 187행)의 여러 테스트가 **초기 진입 시 그룹이 펼쳐져 자식 노드(`node-docling`/`node-presidio`/`node-kure`)가 DOM에 존재**함을 전제한다. "기본 접힘"으로 바꾸면 아래 테스트가 **깨진다**:
  - 25행 `docling·presidio·kure가 서로 다른 x좌표로 렌더` — `CHILD_IDS` 자식 노드 미부착 → `not.toBeNull()` 실패
  - 58행 `그룹 자식 nodeclick → 노드 상세 모달` — 자식 노드 미존재 → `clicked` null → 실패
  - 73행 `그룹 collapse 버튼 클릭 → 자식 미부착 + 재클릭 복원` — 이미 접힘이라 `.airflow-collapse-btn`(펼침 상태의 ▾) 부재 → 실패
  - 113행 `E-1: collapsed 후 경계 엣지` — collapse 버튼 클릭 전제가 무너짐 → 실패
- 따라서 계획 원문의 **"기존 e2e 10/10 회귀 없음"은 성립하지 않는다.** 이 테스트들을 "먼저 '모두 펼치기' 클릭 후 자식 검증" 또는 "이미 접힌 상태 반영"으로 **수정**해야 한다(작업 C).
- `beforeEach`(12~17행)의 `.svelte-flow__node` first 부착 대기는 그룹 노드도 flow 노드이므로 그대로 통과 → 수정 불필요.

**② 테스트 하네스·환경 전제**
- 이 spec은 **sample 모드**(`/sample/pipeline`)만 검증(real 모드는 test 환경에서 백엔드 대기 스텁). 그룹 컨테이너 = `node-airflow`, 자식 = docling/presidio/kure.
- `ToolCanvasView`는 Svelte 컴포넌트라 단위테스트 불가 → **e2e로만 검증**. `buildNodesAndEdges` 자체 로직은 변경 없음(기존 단위테스트 회귀만 확인).

**③ 실행 순서·동일 파일 편집 충돌·병렬 가능 여부**
- **A·B는 동일 파일(`ToolCanvasView.svelte`)** → 반드시 **하나의 에이전트**로 묶어 편집.
- **C·D는 동일 파일(`airflow-group.spec.ts`)** → 하나의 에이전트로 묶어 편집. A·B 구현 완료 후 진행(펼치기 버튼 `data-testid` 확정 후 e2e 작성).
- A·B(구현)와 C·D(테스트)는 파일이 다르므로 순차(A·B → C·D)로 위임.

**④ 초기화 `$effect` 무한 루프 주의 (미선택 결정 근거)**
- 초기 접힘 세팅 `$effect`는 **`topology`에만 의존**하도록 격리한다. 기존 43행 build `$effect`(`collapsedGroups` 의존)와 합치면 `collapsedGroups` 재대입 → 재실행 루프 위험.
- 사용자가 이미 펼친 그룹을 보존하지 **않고** topology 교체 시 전부 리셋한다(선택). 근거: topology 교체 = 뷰 전환 수준 변화 → 상태 리셋이 자연스러움. (탈락: 펼침 상태 보존 — topology 노드 집합이 바뀌면 보존 대상 매칭이 모호해 복잡도만 증가)
