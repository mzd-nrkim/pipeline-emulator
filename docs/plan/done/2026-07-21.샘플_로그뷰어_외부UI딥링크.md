# 샘플 파이프라인 페이지 — 로그 뷰어 + 외부 UI 딥링크

> 상태: 통테통과-완료

데모용 `/sample/pipeline` 페이지에 시연 설득력을 높이는 두 기능을 최소 비용으로 추가한다. 현재 샘플 페이지는 노드 드로어에 "노드 상세 / 실행 이력" 2탭과 트리거·런타임 설정 폼을 갖췄으나, ① 실행 로그를 볼 수단이 없고 ② 실제 Airflow/NiFi UI로 진입하는 동선이 없다.

조사 결과 인프라는 대부분 이미 존재한다:
- ui-backend가 `/var/run/docker.sock`을 마운트하고 docker SDK 사용(컨테이너 로그 인프라 有), 서비스 전원제어·헬스수집·Airflow REST 연동 구현됨.
- Airflow UI는 `localhost:8080`, NiFi UI는 `localhost:8443`(nifi 프로필)에 이미 노출됨.
- 노드는 `config.dagId`(Airflow 계열)와 `tool` id(예: `apache-nifi`)를 보유.

단, `/sample/pipeline`은 **mock 어댑터**를 쓰므로 백엔드를 호출하지 않는다. 사용자 확정에 따라 **로그는 sample 모드에서 mock 가짜 로그로 표시**하고, **실행/UI 기능은 외부 UI 딥링크 버튼**으로 구현한다.

---

## 목표

- 샘플 페이지 노드 드로어에 **"로그" 탭**이 추가되어, 선택 노드의 실행 로그를 mock 데이터로 표시한다.
- 로그 소스를 **컨테이너 로그 / Airflow 태스크 로그** 2종으로 토글할 수 있다.
- 로그 패널이 **수동 새로고침 + 자동 폴링(2초)** 토글을 제공해 "실시간" 느낌을 낸다.
- 노드 드로어에 **외부 UI 딥링크 버튼**이 노출되어, 해당 노드의 Airflow DAG grid(`localhost:8080/dags/{dagId}/grid`) 또는 NiFi UI(`localhost:8443/nifi`)를 새 탭으로 연다.
- 두 기능 모두 mock/real 어댑터 인터페이스 정합성을 깨지 않고, 타입 체크·단위테스트를 통과한다.

## 접근 방법

1. **타입 계약 확장** (`frontend/src/lib/api/types.ts`)
   - `LogSource = 'container' | 'airflow'`, `LogLevel`, `LogLine { ts; level; message }`, `LogResponse { source; nodeId; lines }` 추가.

2. **로그 생성기 모듈 신설** (`frontend/src/lib/mock/logs.ts`)
   - `generateLogs({ nodeId, tool, source, dagId?, tail?, now? }): LogResponse` 순수 함수.
   - `tool` id로 flavor를 매핑(mysql/es/nifi/airflow/valkey/debezium/app/generic)해 서비스별 그럴듯한 컨테이너 로그 라인을 생성.
   - `source='airflow'`는 `dagId`·태스크 id를 참조하는 태스크 인스턴스 로그 형태로 생성.
   - `now`·seed 기반으로 결정론적 생성(테스트 가능) + 호출 시각에 따라 라인이 갱신되어 폴링 시 살아있는 것처럼 보이게 함.

3. **어댑터 인터페이스 정합** (`mock-adapter.ts`, `real-adapter.ts`)
   - 두 어댑터에 `fetchLogs(nodeId, source, tail): Promise<LogResponse>` 추가.
   - mock: `generateLogs` 위임. real: 동일 생성기 재사용(향후 백엔드 `/logs` 엔드포인트 배선은 TODO 주석으로 남김 — 이번 스코프 제외). `real-adapter`가 `../mock/*`를 import하는 것은 기존 `realTopology` 사례와 동일 관례.

4. **외부 UI 링크 헬퍼 신설** (`frontend/src/lib/canvas/externalLinks.ts`)
   - `AIRFLOW_UI_BASE`(`http://localhost:8080`)·`NIFI_UI_BASE`(`https://localhost:8443`) 상수.
   - `getExternalLinks(node): { label; href }[]` — `config.dagId` 있으면 Airflow grid 딥링크, `tool==='apache-airflow'`면 Airflow 홈, `tool==='apache-nifi'`면 NiFi 딥링크 반환.

5. **드로어 UI 확장** (`frontend/src/routes/[mode=mode]/pipeline/+page.svelte`)
   - `drawerTab` 유니온에 `'logs'` 추가, 탭 버튼 추가(노드 선택 시 활성).
   - 로그 탭: 소스 토글, 새로고침 버튼, 자동 폴링 토글, 모노스페이스 로그 패널.
   - 로그 상태(`logSource`·`logLines`·`logLoading`·`logError`·`logAuto`)와 `loadLogs()` 및 폴링 `$effect` 추가.
   - 노드 상세 탭 조작 영역에 `getExternalLinks(selectedNode)` 기반 외부 UI 버튼(`target=_blank rel=noopener`) 렌더.

6. **단위테스트 추가**
   - `frontend/src/lib/mock/logs.test.ts` — 소스별 라인 수·형식·결정론 검증.
   - `frontend/src/lib/canvas/externalLinks.test.ts` — dagId/nifi/airflow 분기별 링크 생성 검증.

## 실행 시 필수 고려사항

> plan-review 검토 중 발견 사항 — plan-run 실행 전 반드시 숙지.

**① 회귀 이유·범위 (낮음, 가법적 변경)**
- 이 변경은 신규 타입·신규 모듈(`mock/logs.ts`, `canvas/externalLinks.ts`)·신규 탭 추가로 대부분 가법적이다. 기존 `drawerTab` 유니온(`'node' | 'history'`)에 `'logs'`를 **추가**하는 것이므로 기존 두 탭 분기(288·320·474행)는 건드리지 않는다.
- 유일한 회귀 위험: `+page.svelte`의 탭 버튼 그룹(262·270행) 마크업에 세 번째 버튼을 삽입할 때 기존 버튼 레이아웃/조건(`disabled`)을 훼손하지 않아야 한다.

**② 테스트 하네스·환경 전제**
- `generateLogs`·`getExternalLinks`는 **순수 함수**로 설계되어 Vitest 단위테스트가 DB·앱 기동 없이 가능하다(pre-gate). 결정론 검증은 `now`(고정 timestamp)를 주입해 수행한다 — `now` 미주입 시 시각 기반 라인 갱신 때문에 스냅샷이 흔들린다.
- sample 모드(`/sample/pipeline`)는 `currentAdapter = page.params.mode === 'real' ? realAdapter : mockAdapter`(15행)에 따라 **mockAdapter**를 사용하므로 로그 탭은 `mock-adapter.fetchLogs` → `generateLogs` 경로로 동작한다.

**③ 실행 순서·동일 파일 편집 충돌·병렬 가능 여부**
- **A(types.ts)·B(mock/logs.ts)·D(canvas/externalLinks.ts)** 는 독립 파일 → 병렬 위임 가능.
- **C(어댑터)** 는 A·B 완료 후(타입·generateLogs 참조). **E(+page.svelte)** 는 A·C·D 완료 후.
- **E-1~E-4는 모두 동일 파일(`+page.svelte`)** 편집 → 반드시 **하나의 에이전트**로 묶어 위임(동시 편집 충돌 방지).
- **F(단위테스트)** 는 B·D 완료 후.

**④ 미선택 결정 근거**
- sample 모드 로그는 **mock 가짜 로그**로 표시(사용자 확정). real 백엔드 `/logs` 엔드포인트 배선은 이번 스코프 **제외** — `real-adapter.fetchLogs`는 동일 `generateLogs`를 재사용하고 `// TODO: 백엔드 /logs 배선` 주석만 남긴다. (탈락 옵션: 실제 컨테이너 로그 스트리밍 — 백엔드 배선·SSE 필요로 비용 과다)
- `real-adapter`가 `../mock/logs`를 import하는 것은 기존 `realTopology`(59행) 사례와 동일 관례 → 정합성 OK.
