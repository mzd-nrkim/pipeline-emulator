# infra 뷰 mysql 소스 라벨 오표시 교정 — per-node displayName override

> 상태: 통테통과-완료
> 분리 출처: `infra_뷰_의존성_배선_교정.md`의 G-4/D-4에서 분리(사용자 확정 2026-07-20). 통합 초안(`infra뷰_잔여교정_kibana허위의존_mysql소스라벨.md`)에서 G-4만 격리·독립.

infra 뷰에서 `node-mysql-container`(원본 소스 DB) 카드 제목이 **"MySQL (Silver/Gold)"** 로 오표시된다. 카드 제목은 `node.label`이 아니라 **카탈로그 `displayName`** 을 쓰는데(`ToolFlowNode.svelte:10,65` ← `buildNodesAndEdges.ts:194` ← `toolCatalog.ts:217`), `node-mysql-container`가 `tool:'mysql'`이라 sink용 라벨을 그대로 상속한다. 즉 **소스 DB를 silver/gold 싱크처럼 라벨링**하는 결함이다.

근본 원인은 `node-mysql`(sink, "Silver/Gold" 라벨이 **정확**)과 `node-mysql-container`(source_db, 오표시)가 카탈로그 `mysql` **한 엔트리를 공유**하는 것이다. 따라서 카탈로그를 고치면 정상인 sink까지 깨진다 → **카탈로그 불변, per-node override가 정답**.

> 독립성: 본 계획은 `deployStatus`(P)·`es→kibana`(kibana 계획)와 **별 축**(카탈로그/라벨)이라 무관·독립 병렬 가능. 단 `topology.ts`를 만지므로 다른 topology 편집 계획과는 편집 라인만 직렬화.

---

## 배경: 실측 근거

| 항목 | 값 | 근거 |
|------|----|----|
| 카드 제목 소스 | `data.displayName`(카탈로그값) — `node.label` 미사용 | `ToolFlowNode.svelte:10,65`, `buildNodesAndEdges.ts:194,208` |
| 카탈로그 `mysql` | `displayName: 'MySQL (Silver/Gold)'` | `toolCatalog.ts:217` |
| 오표시 노드 | `node-mysql-container`(`tool:'mysql'`, `source_db`) — sink 라벨 상속 | `topology.ts:204` |
| 정상 노드 | `node-mysql`(sink, Silver/Gold 라벨 정확) — 카탈로그 공유 | `topology.ts:190` |
| 미사용 필드 | `node-mysql-container.label = 'MySQL 원본 DB'`(렌더에 안 쓰임) | `topology.ts:207` |

## 목표

- infra 뷰에서 `node-mysql-container` 카드 제목이 소스 DB 성격(예: "MySQL 원본 DB")으로 표시된다.
- `node-mysql`(sink)의 "MySQL (Silver/Gold)"는 **불변**이다.
- 카탈로그 `mysql` 엔트리·기타 전 노드 라벨·`data` 채널·`FlowNode.data` shape·핸들 계약은 **불변**. override 미지정 노드 회귀 0.

## 접근 방법

1. **override 필드 추가** (`$lib/api/types.ts`): `ToolNode`에 `displayNameOverride?: string` 옵셔널 필드. 미지정 노드는 카탈로그 경로 그대로 → 회귀 0.
2. **`node-mysql-container`에 override 지정** (`topology.ts:204` 노드): `displayNameOverride: 'MySQL 원본 DB'`(사용자 확정 라벨) 추가.
3. **`buildNodesAndEdges`가 override 우선** (`buildNodesAndEdges.ts:191~217`): `displayName = n.displayNameOverride ?? entry.displayName`으로 계산해 `catalogData`·`label` 조립(208행) 모두 반영. vendor/icon/accent는 카탈로그 유지.

## 실행 시 필수 고려사항

### override는 노드 단위 → sink 회귀 0

`node-mysql`(sink)과 `node-mysql-container`(source)가 카탈로그 한 엔트리를 공유하는 게 버그의 근본. override는 노드 단위라 `node-mysql-container`만 라벨이 바뀌고 `node-mysql`은 카탈로그값 유지 → sink 라벨 회귀 0. **카탈로그 엔트리는 절대 수정하지 않는다.**

### override 미사용 노드 회귀 0

`displayNameOverride`는 옵셔널·`node-mysql-container` 1개 노드에만 지정, `?? entry.displayName` 폴백 → 나머지 전 노드 기존 경로 그대로. 회귀 표면은 mysql-container 카드 제목 1곳뿐.

### 렌더 계약 불변

`ToolFlowNode`는 `data.displayName`을 읽을 뿐(10·65행) → 수정 불필요. buildNodesAndEdges가 주입하는 값만 바뀐다. 핸들·채널 필터·카드 구조 셀렉터 유지.

### deployStatus 계획과 동일 파일 편집 충돌 (병렬 금지·직렬화)

intro의 "deployStatus와 무관·독립 병렬 가능"은 **로직 축의 독립성**을 뜻하며 **파일 편집은 공유**된다 — plan-review 실측 결과 두 계획은 같은 파일 두 곳을 편집한다:

- `frontend/src/lib/api/types.ts` — 본 계획 G4-1이 `ToolNode` 인터페이스(100~109행)에 `displayNameOverride?`, deployStatus S-1이 동일 인터페이스에 `deployStatus?` 추가 → 인접 라인 충돌.
- `frontend/src/lib/canvas/buildNodesAndEdges.ts` — 본 계획 G4-3이 노드 data 조립(catalogData 191~195·label 208행), deployStatus S-2가 같은 data 블록(FlowNode.data 213행 인근) 편집 → 같은 `data: {...}` 리터럴 충돌 위험.

따라서 두 계획은 **병렬 워크트리 금지**다. 순서는 무관하나 **직렬 실행**(한쪽 머지 후 다른 쪽 리베이스)해야 한다. plan-run에 두 계획을 한 배치로 넣으면 동일 파일 편집으로 직렬 배치된다.

### TC 커버리지 검토 (Right-BICEP · CORRECT)

카탈로그 공유 노드의 per-node override라 TC는 **override 분기(대상/비대상) + 카탈로그 불변**에 집중.

**Right-BICEP**
- **Right(정상)**: `node-mysql-container` 카드 제목 = override값, `node-mysql` = 카탈로그값.
- **B(경계)**: override 미지정 노드(전량) = 카탈로그 폴백, 빈 override(`''`)는 falsy → 카탈로그 폴백(의도 확인).
- **I(역/부정)**: 카탈로그 `mysql.displayName` 불변 부정 단언(sink 회귀 방지).
- **C(교차검증)**: 같은 `tool:'mysql'` 두 노드가 서로 다른 카드 제목을 갖는지 교차 확인.
- **E(에러)**: 해당 없음 — 정적 데이터·문자열 폴백.
- **P(성능)**: 해당 없음 — 문자열 1회 `??` 분기.

**CORRECT**
- **Conformance**: override 필드 타입(옵셔널 string), 폴백 규칙(`?? entry.displayName`).
- **Reference**: `FlowNode.data` shape·렌더 계약 불변(data.displayName 값만 변경).
- **Existence**: override 미전달 시 카탈로그 폴백(안전).
- **Cardinality**: 노드·엣지 수 불변(라벨값만 변경).
- 나머지(Ordering/Range/Time): 해당 없음 — 라벨은 순서·좌표·시간 무관.

**e2e**: 기존 `infra-view.spec.ts` 회귀 통과 + mysql-container 카드 제목 단언 보강.
**DB 통합**: 해당 없음 — mock 토폴로지.
**teardown**: 해당 없음.
