# 고아 노드 숨김 토글 옵션 — 캔버스 뷰에 "연결 없는 노드 표시" 스위치 추가

> 상태: 미시작

현재 `buildNodesAndEdges`는 현재 뷰(data/infra)의 채널로 엣지를 거른 뒤 **엣지가 하나도 없는 노드(고아)를 무조건 숨긴다**(`buildNodesAndEdges.ts:124-130`, 주석 "고아 노드 숨김"). 이는 각 뷰를 깔끔하게 유지하지만 **강제**라 사용자가 전체 노드를 볼 수 없다. 본 계획은 이 숨김을 **끄고 켤 수 있는 UI 토글**로 전환한다. 기본값은 현행 동작(숨김)을 유지하고, 사용자가 토글하면 연결 없는 노드까지 모두 표시한다.

> 별개 계획 `2026-07-20.infra_뷰_의존성_배선_교정.md`(dependency 엣지 정합)와 **독립**이다. 두 계획 모두 `buildNodesAndEdges.ts`를 건드리나 편집 영역이 다르다(교정=LAYER_MAP/ORDER·엣지 데이터, 본 계획=시그니처 param·필터 분기). 실행 순서 권장: **infra 교정 → 본 토글**(교정이 먼저 머지되면 토글 ON 시 노드/엣지가 이미 정확). 순서 뒤바뀌어도 충돌은 동일 파일 다른 라인이라 병합 가능.

## 목표

- 캔버스 뷰에 **"연결 없는 노드 표시" 토글**(체크박스/버튼)이 뷰 전환 토글(data/infra) 근처에 노출된다.
- 토글 OFF(기본) = 현행 동작(고아 숨김) 그대로. 토글 ON = 현재 뷰에서 엣지 0인 노드까지 모두 렌더.
- 토글 상태는 **data↔infra 뷰 전환 간에 유지**된다(뷰 바꿔도 스위치 값 보존).
- 고아 노드가 표시될 때도 **유효한 좌표로 배치**되어 다른 노드와 심하게 겹치지 않는다.
- 기존 렌더 계약(`FlowNode.data` shape·핸들·채널 필터)·data 채널·카탈로그는 불변. 기본 OFF이므로 **기존 e2e·시각 회귀 0**.

## 접근 방법

1. **`buildNodesAndEdges`에 `hideOrphans` 파라미터 추가** (`buildNodesAndEdges.ts`):
   - 시그니처를 `buildNodesAndEdges(topo, view='data', hideOrphans=true)`로 확장(기본 `true` → 기존 호출부 무변경 시 현행 유지).
   - 고아 필터(124~130행)를 조건부로: `hideOrphans`이면 현행 `connectedIds` 필터, 아니면 `visibleNodes = topo.nodes`(전체). 위치 계산(`computeDepths`/`computeInfraPositions`)은 이미 전달된 노드 전체에 좌표를 부여하므로(data=미방문 fallback depth `maxDepth+1`, infra=`getInfraLayer`→`other`) 별도 배치 로직 불필요 — 정적 확인만.

2. **`ToolCanvasView`에 `hideOrphans` prop 추가·전달** (`ToolCanvasView.svelte`):
   - props에 `hideOrphans: boolean = true` 추가(14행 props 블록), `$effect`의 호출을 `buildNodesAndEdges(topology, view, hideOrphans)`로(27행).
   - `$effect` 의존성에 `hideOrphans`가 포함돼 토글 변경 시 재계산되는지 확인(Svelte 5 runes는 자동 추적).

3. **페이지에 토글 상태·UI 추가** (`src/routes/[mode=mode]/pipeline/+page.svelte`):
   - `let showOrphans = $state(false);`(21행 `view` 상태 근처) — 기본 false = 숨김(현행).
   - data/infra 뷰 토글 버튼(129~142행) 옆에 "연결 없는 노드" 토글 버튼/체크박스 추가(동일 스타일 톤).
   - `<ToolCanvasView>`(151행)에 `hideOrphans={!showOrphans}` 전달.

4. **스코프 한정**: 고아 숨김은 **`ToolCanvasView`가 쓰는 공용 `buildNodesAndEdges.ts`에만** 존재한다. `PipelineGraphView.svelte`는 자체 로컬 `buildNodesAndEdges`(stage 기반, 24행)라 고아 숨김 로직이 없음 → 본 토글 대상 아님. 토글은 tool 캔버스(data/infra)에만 적용.

## 실행 시 필수 고려사항

### 기본값 = OFF(숨김) → 기존 회귀 0

토글 기본값을 `showOrphans=false`(= `hideOrphans=true`)로 두면 기존 호출·렌더가 100% 동일하다. 따라서 기존 e2e(`infra-view`·`pipeline-canvas`·`node-detail-modal`)·시각 스냅샷은 **기본 상태에서 회귀 0**. 신규 검증은 토글 ON 경로에만 필요.

### 고아 노드 좌표 안정성

`hideOrphans=false`로 전체 노드를 위치 함수에 넘길 때:
- **data 뷰**: `computeDepths`가 미방문 노드에 `maxDepth+1` 폴백 depth를 준다(277~280행) → 고아들이 우측 끝 열에 세로로 쌓임. 겹침은 `depthCounters` 세로 분배로 회피되나, **여러 고아가 같은 depth에 몰리면 세로로 길게 늘어설 수 있음** → Z에서 실측(겹침·화면 밖 이탈 확인).
- **infra 뷰**: 매핑 없는 노드는 `getInfraLayer`→`other` 계층(맨 아래)로 모임. 매핑된 고아(예: dam=매핑 없음, s3-bronze 등)는 각자 계층. `computeInfraPositions`가 계층 내 순번 x 분배 → 겹침 없음.
- 결론: 별도 레이아웃 코드 없이 기존 폴백으로 배치 가능. 단 data 뷰 고아 다수 시 시각적 정렬은 Z-post 브라우저에서 확인하고, 필요하면 후속 개선(본 계획은 "표시 가능·겹침 없음"까지가 목표).

### 토글 상태 뷰 전환 간 보존

`showOrphans`는 `view`와 별개 `$state`라 뷰 토글(data↔infra) 시 값이 유지된다. 두 상태가 한 컴포넌트에 있으므로 리셋 로직을 넣지 않는 한 자동 보존 — 별도 작업 불필요, 검증만.

### 접근성

토글 버튼은 `aria-pressed`로 on/off 상태를 노출하고, 라벨("연결 없는 노드 표시" 등)을 텍스트로 둔다. 기존 뷰 토글 버튼(131·138행)의 `aria`·스타일 패턴을 그대로 따른다.

### infra 교정 계획과의 관계

`infra_뷰_의존성_배선_교정.md`가 `buildNodesAndEdges.ts`의 LAYER_MAP/ORDER·topology 엣지를 바꾼다. 본 계획은 같은 파일의 **시그니처·필터 분기**만 손대 편집 영역이 겹치지 않는다. 두 계획을 병렬 실행하면 동일 파일이라 머지 충돌 가능 → **직렬 권장(교정 먼저)**. plan-run 시 두 계획을 한 배치로 넣으면 의존성 순서로 직렬 배치된다.

### Node 정적/e2e 게이트 위치 (Z-post)

`frontend/package.json` 의존(`npm run check`/`build`/`vitest`/`playwright`)은 gitignored `node_modules` 필요 → 워크트리(Z-pre) 신뢰 불가 → **머지 후 원본 main(Z-post)** 실행. DB·마이그레이션 변경 없음(순수 프론트 상태·렌더).

## Verification

- 캔버스에 "연결 없는 노드 표시" 토글이 노출되고, OFF(기본)에서 기존과 동일하게 고아가 숨겨진다.
- 토글 ON 시 현재 뷰의 엣지 0 노드가 유효 좌표로 표시되고 다른 노드와 겹치지 않는다.
- 토글 상태가 data↔infra 뷰 전환 간 유지된다.
- 기본값 OFF이므로 기존 e2e·시각 스냅샷 회귀 0, ToolFlowNode·data 채널·카탈로그 불변.
- `npm run check`·`build`·`vitest`·`playwright`가 수치 근거(통과 카운트)와 함께 통과한다.

### TC 커버리지 검토 (Right-BICEP · CORRECT)

UI 토글 + 렌더 필터 분기이므로 TC는 **토글 on/off 렌더 차이 + 기본값 회귀 + 뷰 전환 보존**에 집중한다.

**Right-BICEP**
- **Right(정상)**: 토글 ON → 고아 노드 표시, OFF → 숨김(단위 `buildNodesAndEdges(topo, view, false)` 노드 수 > `true` 노드 수).
- **B(경계)**: 고아 0인 뷰(토글 효과 없음), 모든 노드가 고아인 극단(빈 엣지 topology) — 토글 ON 시 전체 표시·OFF 시 전부 숨김.
- **I(역/부정)**: `hideOrphans=true` 기본값이 기존 `connectedIds` 필터와 **완전 동일** 결과(회귀 부정 단언).
- **C(교차검증)**: 브라우저에서 토글 ON 시 data·infra 각 뷰의 실제 고아 노드 집합이 예상과 일치.
- **E(에러)**: 해당 없음 — 렌더 필터 분기, 예외 경로 없음(빈 topology는 경계에서 커버).
- **P(성능)**: 해당 없음 — 노드 수 소폭 증가, 렌더 비용 무시 가능.

**CORRECT**
- **Conformance**: `hideOrphans` boolean 두 값 각각의 노드 집합.
- **Ordering**: 해당 없음 — 노드 렌더 순서 무관.
- **Range**: 고아 노드 좌표가 캔버스 범위 내(data fallback depth·infra `other` 계층) — 화면 밖 이탈·겹침 0.
- **Reference**: `FlowNode.data` shape·핸들·채널 필터 계약 불변, `PipelineGraphView` 무관.
- **Existence**: `hideOrphans` 미전달 시 기본 `true`(기존 호출부 안전), 빈 topology 크래시 없음.
- **Cardinality**: 토글 ON 노드 수 = 전체 topo.nodes, OFF 노드 수 = connectedIds 부분집합. ON ≥ OFF.
- **Time**: 토글 클릭 → `$effect` 재계산으로 즉시 반영, 뷰 전환 간 `showOrphans` 상태 보존.

**e2e**: 신규 `orphan-toggle.spec.ts`(Z-post) — 토글 ON/OFF 노드 수 차이·뷰 전환 보존. 기존 스펙은 기본 OFF라 회귀 0.
**DB 통합**: 해당 없음 — 순수 프론트 상태·렌더, DB·마이그레이션 무관.
**teardown**: 해당 없음 — 생성·저장 레코드/파일 없음(UI 상태만).
