# 파이프라인 에뮬레이터 — Design Prompt

> 대상: 데이터 파이프라인 에뮬레이터 전체 화면 (모니터링 대시보드 · 실행 조작 · 문서 드릴다운 · 검색 데모 · 설정 · 컴포넌트 라이브러리)
> 근거: [pipeline-emulator-decisions.md](./pipeline-emulator-decisions.md) · [lodestar-reuse-assessment.md](./lodestar-reuse-assessment.md) · [pipeline-emulator-sample-data-plan.md](./plan/pipeline-emulator-sample-data-plan.md)
> 스타일: 색상·타이포·간격은 구현 도구(Lovable 등)의 테마 설정에 위임한다. 브랜드 색상이 확정되지 않아 `DESIGN.md`를 상류에서 미리 고정하지 않으며, 구현 결과의 팔레트가 정해지면 그때 `DESIGN.md`로 역캡처한다. 이 문서는 기능 요구사항만 기술한다.
> 상태 색상은 값이 아니라 의미(완료·진행·대기·실패·없음)로만 규정하며, 관습적 매핑은 구현 도구에 맡긴다.
> 범위 주: MVP 종착은 Gold(파이프라인 필드매핑)까지다. 검색 데모·정밀 마스킹 등 MVP 이후 축도 설계에 포함하되, 제공되지 않는 동안은 예정 상태로 노출한다.

## 페이지 탭

- **개요** — 에뮬레이터가 재현하는 파이프라인 전체 그림과 데모 시작
- **파이프라인** — 단계별 처리 현황 실시간 모니터링 + 실행 조작
- **문서** — 개별 문서의 단계별 변화와 마스킹 전후 비교
- **검색** — 검색 데모(질의·결과·하이브리드 스코어)
- **설정** — 에뮬레이터 구성 축 토글
- **컴포넌트** — 재사용 화면 요소 모음

---

## 개요 페이지 요구사항

### 원칙

- 데모의 도입부다 — 처음 보는 사람이 이 에뮬레이터가 무엇을 재현하는지 이해할 수 있다.
- 모바일 우선이며, 전체 그림을 한 화면에서 파악할 수 있다.

### 내용

- 재현 대상 파이프라인의 전체 여정이 요약되어 보인다: 데이터 투입 → 3계층(Bronze · Silver · Gold) 6단계 처리 → 검색.
- 데모 내러티브 단계가 순서로 보인다: 투입 → 단계 통과 → 최종 적재 → 검색.
- 현재 구성 요약이 보인다 (어떤 구성 축이 켜져 있는지 — 예: 마스킹 방식, 검색 서빙 상태).
- 데모를 시작할 수 있다 (샘플 데이터 투입과 파이프라인 실행으로 진입).
- 원본 파이프라인과 에뮬레이터 대체 구성의 대응이 보인다 (각 단계·컴포넌트가 실제 시스템의 무엇을 대체·축소하는지).
- 각 주요 화면(파이프라인 · 검색 · 설정)으로 이동할 수 있다.

## 파이프라인 페이지 요구사항

### 원칙

- 처리 현황 표시는 읽기전용이다 — 표시 요소의 조작이 처리 결과를 바꾸지 않는다.
- 단, 실행 조작 영역에서 데이터 투입·실행 시작·재실행을 할 수 있다.
- 별도 새로고침 없이 최신 처리 현황이 실시간으로 반영된다.
- 아직 투입·실행 전인 상태가 구분되어 보이고, 다음에 할 일(데이터 투입 · 실행 시작)이 안내된다.
- 모바일 우선이며, 좁은 화면에서도 전체 흐름을 파악할 수 있다.

### 실행 조작

- 샘플 데이터를 투입할 수 있다 (투입 규모·PII 밀도 등 조건을 정할 수 있다).
- 파이프라인 실행을 시작할 수 있다.
- 전체 또는 특정 단계부터 재실행할 수 있다.
- 실행 중에는 진행 상태가 보이고, 진행 중인 실행을 멈출 수 있다.
- 조작 가능 여부가 현재 실행 상태에 따라 노출된다 (실행 중에는 시작이 비활성으로 보이는 등).
- 실행이 완료되거나 실패하면 그 결과가 알림으로 드러난다.

### 파이프라인 흐름

- 데이터 성숙 3계층(Bronze · Silver · Gold)이 흐름 순서로 보인다.
- 6개 처리 단계가 흐름 순서로 연결되어 보인다: 브론즈 등록 → 실버 구조화 → 실버 마스킹 → 골드 청킹 → 골드 엔리치먼트 → 골드 필드매핑.
- 각 단계가 어느 계층에 속하는지 구분되어 보인다.
- 단계 사이의 데이터 이동 방향이 보인다.
- 전체 파이프라인이 어느 단계까지 진행됐는지 한눈에 보인다.
- 흐름을 확대·축소하거나 이동해 원하는 단계를 볼 수 있다.

### 단계 노드

각 처리 단계에 다음이 보인다:

- 단계 이름과 소속 계층
- 처리 상태 (완료 · 진행 중 · 대기 · 실패 · 데이터 없음)
- 그 단계를 통과한 문서 수
- 마지막으로 처리된 시각
- 그 단계가 하는 일에 대한 짧은 설명
- 단계를 선택하면 해당 단계의 세부 정보를 볼 수 있다.

### 단계 세부

선택한 단계에서 다음이 보인다:

- 입력 문서 수와 출력 문서 수 (단계를 지나며 문서 수가 늘거나 유지되는 변화)
- 최근 실행의 시작 시각, 소요 시간, 성공·실패 여부
- 실패한 경우 실패 사유
- 그 단계의 문서 목록으로 이동해 개별 문서를 볼 수 있다.

### PII 마스킹 현황

실버 마스킹 단계를 선택하면 추가로 다음이 보인다:

- 마스킹된 문서 수와 마스킹되지 않은 문서 수
- 탐지된 PII 유형별 건수 (전화 · 주민번호 · 이메일 · 계좌 등)
- 현재 적용 중인 마스킹 방식
- 현재 방식에서 아직 가려지지 않는 PII 유형은 예정 표시로 구분되어 보인다.

### 검색 단계

- 검색 단계가 파이프라인 흐름의 끝에 보인다.
- 검색 서빙이 제공되지 않는 동안은 예정 표시로 구분되어 보이고, 제공될 때는 검색 페이지로 이동할 수 있다.

### 실행 이력

- 파이프라인 실행 회차 목록이 보인다.
- 각 실행에 실행 식별자, 시작 시각, 전체 소요 시간, 최종 결과(성공 · 실패 · 진행 중)가 보인다.
- 각 실행이 어떤 구성(마스킹 방식 · 검색 서빙 등 토글 상태)으로 실행됐는지 보인다.
- 실행을 선택하면 그 회차의 단계별 결과를 볼 수 있다.
- 두 실행을 골라 단계별 결과를 비교할 수 있다 (예: 정규식 마스킹 실행과 정밀 마스킹 실행의 차이).

## 문서 페이지 요구사항

### 원칙

- 파이프라인이 개별 문서를 어떻게 바꾸는지 실물로 확인할 수 있다.
- 마스킹 전후 비교가 이 화면의 핵심 시연 요소다.

### 문서 목록

- 파이프라인을 통과한 문서 목록이 보인다.
- 각 문서에 식별자, 원본 중요도, 현재 도달 단계, 보안 분류, 마스킹 여부가 보인다.
- 조건(중요도 · 보안 분류 · 마스킹 여부 · 도달 단계)으로 목록을 좁혀 찾을 수 있다.
- 문서를 선택하면 단계별 변화를 볼 수 있다.

### 단계별 변화

선택한 문서에서 단계별로 다음이 보인다:

- 브론즈 단계의 원본 레코드(수집된 원 데이터)가 변환 전 상태로 보인다.
- 문제에서 부품 · 차종 · 프로젝트로 갈라지는 관계가 보인다 (단계마다 문서 수가 늘어나는 근거).
- 구조화 결과 (문제 본문 · 부품 · 처리 단계 등 구조화된 내용)
- 마스킹 전 텍스트와 마스킹 후 텍스트가 함께 비교되어 보인다.
- 가려진 PII의 위치와 유형이 표시되어 보인다.
- 마스킹 방식(정규식 · 정밀)에 따라 가려지는 PII 유형이 달라지는 것이 보인다 (정밀 방식이 아닌 동안 이름·주소 등은 가려지지 않음이 드러난다).
- 청킹 결과 (분할된 청크들과 순서)
- 엔리치먼트 결과 (키워드 · 요약 · 개체명)
- 필드 매핑 결과 (검색계 인수용 정보 — 대상 인덱스 · 라우팅 · 보안 분류)

## 검색 페이지 요구사항

### 원칙

- 검색 서빙이 켜졌을 때 제공된다.
- 검색 서빙이 꺼져 있는 동안은 예정임이 안내되고, 켜는 경로가 함께 보인다.

### 질의

- 검색어를 입력해 질의할 수 있다.
- 검색 방식(키워드 · 의미 · 하이브리드)을 선택할 수 있다.

### 결과

- 결과 목록에 각 문서의 제목, 요약, 중요도, 보안 분류가 보인다.
- 각 결과에 관련도 점수가 보인다.
- 하이브리드 방식일 때는 키워드 기여와 의미 기여가 각각 보인다.
- 각 결과에서 질의와 일치한 구간이 강조되어 보인다.
- 결과를 조건(보안 분류 · 중요도 · 차종)으로 좁힐 수 있다.
- 결과를 선택하면 해당 문서의 단계별 변화(문서 페이지)로 이동할 수 있다.
- 결과가 없으면 없음이 안내된다.
- (예정) 보안 분류·권한에 따라 접근 가능한 결과가 달라지는 것을 시연할 수 있다.

## 설정 페이지 요구사항

### 원칙

- 에뮬레이터 구성 축을 각각 켜고 끄거나 값을 바꿀 수 있다.
- 아직 제공되지 않는 축은 조작할 수 없고, 예정임이 드러난다.

### 구성 축

다음 구성 축이 각각 보이고, 값을 선택할 수 있다:

- 실행 방식 (단일 · 분산)
- 수집기 (스크립트 · 수집 도구)
- 실시간 변경 감지 (끔 · 켬)
- 검색 서빙 (끔 · 경량 · 하이브리드)
- 청킹·엔리치먼트 (모의 · 실제)
- PII 마스킹 (정규식 · 정밀)
- 검색 노드 구성 (단일 · 클러스터)

각 구성 축에 다음이 보인다:

- 축 이름과 현재 선택된 값
- 선택 가능한 값들
- 그 축이 무엇을 바꾸는지에 대한 짧은 설명
- 아직 제공되지 않는 축은 예정 표시와 함께 비활성으로 보인다.

### 조작

- 제공되는 축은 값을 바꿀 수 있다.
- 예정 축은 값을 바꿀 수 없고, 예정임이 안내된다.
- 검색 노드 구성 축은 검색 서빙이 켜져 있을 때만 조작할 수 있다.

## 컴포넌트 페이지 요구사항

### 원칙

- 화면 전반에서 재사용되는 요소들을 모아 확인할 수 있다.
- 각 요소의 여러 상태를 한 화면에서 비교해 볼 수 있다.

### 내용

- 각 재사용 요소에 이름, 용도, 가능한 상태(변형)가 함께 보인다.
- 다음 요소가 포함되어 보인다:
  - 상태 배지 (완료 · 진행 · 대기 · 실패 · 없음 변형)
  - 상태 점
  - 진행 스테퍼 (계층 진행 표시)
  - 단계 노드 (계층 · 상태 · 문서 수)
  - 실행 이력 항목
  - PII 유형별 카운트 표시
  - 마스킹 전후 비교 요소
  - 검색 결과 항목
  - 구성 토글
  - 예정 배지

## 부록 A — 표시 예시 값 (참고)

> 목적: 구현 도구가 도메인에 맞는 목업 콘텐츠를 채우도록 돕는 **참고 예시**다. 값을 강제하는 규범이 아니며, 스타일과 무관하다. 정확한 스키마·규격은 [pipeline-emulator-sample-data-plan.md](./plan/pipeline-emulator-sample-data-plan.md)를 따른다.

- **처리 단계 이름**: 브론즈 등록 · 실버 구조화 · 실버 마스킹 · 골드 청킹 · 골드 엔리치먼트 · 골드 필드매핑
- **단계별 문서 수(소규모 데모)**: 5 → 5 → 5 → 15 → 15 → 15 (청킹부터 문서 수가 늘어남)
- **문서 식별자 형태**: `AP00005928||1`
- **중요도**: S · A · B · C · D · E
- **보안 분류**: RESTRICTED · INTERNAL · PUBLIC (중요도 S→RESTRICTED, A~C→INTERNAL, D~E→PUBLIC)
- **차종 코드**: NX01 등 2~3종
- **PII 예시와 마스킹 결과**:
  - 전화 `010-1234-5678` → `010****1234`
  - 주민번호 `900101-1234567` → `[주민번호 마스킹]`
  - 이메일 `user@hmc.example` → `[이메일 마스킹]`
  - 계좌 `123-456789-12` → `[계좌번호 마스킹]`
  - 이름 `김철수`, 주소 `서울시 강남구` → 정밀 마스킹 방식에서만 가려짐
- **엔리치먼트 예시**: 키워드(명사 상위) · 요약(첫 문장) · 개체명(심어둔 이름·차종)
- **검색 질의 예시**: 도메인 문제/대책 관련 자연어 (예: 부품·현상 키워드)
- **구성 토글 기본값**: 실행 방식=단일 · 수집기=스크립트 · 실시간 변경 감지=끔 · 검색 서빙=끔 · 청킹·엔리치먼트=모의 · PII 마스킹=정규식 · 검색 노드 구성=단일
