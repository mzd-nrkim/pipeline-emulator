# 실행 태스크 인스턴스(/executions) 뷰 연결

백엔드에 실행(run)별 태스크 인스턴스 상세를 반환하는 엔드포인트 `GET /executions?dag_id=X&run_id=Y`(`ui-backend/app/api/executions.py` → `get_task_instances(dag_id, dag_run_id)`, Airflow REST의 `task_instances`를 조회해 태스크별 status를 반환)가 구현돼 있으나, **프론트엔드에 이를 호출하는 어댑터·소비자가 전무**하다(import/fetch 0건). 그 결과 파이프라인 페이지의 "실행 이력" 탭은 run 단위 요약(`stageCounts`)까지만 보여주고, 특정 run을 선택해 **단계/태스크별 진행 상세로 드릴다운할 수 없다**. 백엔드는 있는데 UI가 없는 상태다.

> 스코프 경계: 재실행 버튼·설정 저장·SSE 실시간 진행은 각각 별도 계획서. 본 계획은 선택한 run의 태스크 인스턴스 상세를 조회·표시하는 배선에 집중한다.

---

## 목표

- 어댑터에 `fetchExecutions(dagId, runId)`가 추가되어 백엔드 `/executions`를 호출한다(real: 실제 fetch, mock: 대칭 stub 데이터).
- 파이프라인 "실행 이력" 탭에서 특정 run을 선택하면 해당 run의 **태스크 인스턴스별 상태(status·시각·소요)** 가 표시된다.
- 로딩·빈 결과·오류 상태가 각각 처리된다(무음 실패 금지).
- real 모드는 Airflow에서 실제 태스크 인스턴스를 받아오고, mock 모드는 결정적 stub으로 UI가 동일하게 동작한다.

## 접근 방법

1. **어댑터 신설**: `real-adapter.ts`에 `fetchExecutions(dagId, runId)` — `GET ${BASE}/executions?dag_id=..&run_id=..`, 비정상 응답 시 오류 throw. `mock-adapter.ts`에 대칭 함수 — 주어진 runId 기반 결정적 태스크 인스턴스 목록 반환(6단계 매핑 활용). 타입은 `types.ts`에 `TaskInstance`(taskId/status/startedAt/durationMs 등) 신설 — 백엔드 `get_task_instances` 반환 형태에 맞춤.
2. **소비 지점 결정**: "실행 이력" 탭(`drawerTab === 'history'`)에서 run 항목을 클릭(또는 이미 선택된 `selectedRunId`)하면 해당 run의 `dagId`+`runId`로 `fetchExecutions`를 호출한다. run→dagId 매핑은 `STAGE_DAG_MAP`/토폴로지에서 해석. 최초 진입은 정적 스냅샷, 상세는 온디맨드 로드.
3. **표시 컴포넌트**: 태스크 인스턴스 목록을 렌더하는 소형 뷰(기존 `RunHistoryItem`/`StatusDot` 재사용)를 추가한다. 태스크별 상태 색상은 기존 `statusStroke`/`StatusDot` 규약 재사용.
4. **상태 관리**: `executionsLoading`/`executionsError`/`executions` 상태를 신설해 로딩·오류·빈 결과를 처리. run 선택 변경 시 재조회.
5. **SSE와의 관계**: 실시간 진행(별도 SSE 계획)과 달리, 본 뷰는 **완료/과거 run의 태스크 상세 조회**가 1차 목적이다. 진행 중 run은 SSE 계획이 라이브 반영을 담당하고, 본 뷰는 스냅샷 조회를 담당한다(중복 아님).

## Verification

- **UI 수동 확인**: mock 모드에서 실행 이력의 run을 선택하면 태스크 인스턴스 목록이 상태와 함께 표시된다. real 모드에서 백엔드 기동 시 실제 Airflow 태스크 상태가 표시된다.
- **오류/빈 처리**: 존재하지 않는 run/dag 조회 시 오류 또는 "결과 없음"이 표시되고 페이지가 깨지지 않는다.
- **정적 검증**: `npm run check`(svelte-check) 통과 — `TaskInstance` 타입 정합.
- **백엔드 연동**: `GET /executions?dag_id=..&run_id=..` 200 응답이 UI에 반영된다.
- **회귀 없음**: 실행 이력 요약(`stageCounts`)·run 비교 등 기존 기능 정상.
