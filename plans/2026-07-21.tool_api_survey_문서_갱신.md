# tool-api-survey 문서 갱신 — 누락 도구·통계 정정

`docs/tool-api-survey.md`는 코드(`frontend/src/lib/canvas/toolCatalog.ts`)와 대조 시 (1) **zookeeper·mock-api 2개 도구가 통째로 누락**, (2) 분류 요약 통계가 자체 본문과도 불일치(요약 88 vs 본문 합산 92 vs 실제 코드 101), (3) 하단 "runtime 필드" 서술이 mock-api 3필드를 빠뜨림 — 3가지 문제가 있다. 코드가 정본(source of truth)이므로 문서를 코드에 맞춘다. **이 계획서는 문서(.md)만 수정하며 코드는 건드리지 않는다.**

---

## 목표

- `tool-api-survey.md`의 "도구별 설정 API" 표에 **zookeeper·mock-api 행**이 추가되어, 코드에 정의된 15개 도구가 모두 문서화된다.
- "configField applyMode 분류" 섹션에 **zookeeper(3필드)·mock-api(6필드)** 하위 표가 추가된다.
- "분류 요약" 통계가 **전체 15개 도구 = 101필드 기준(runtime 4 / restart 27 / code 7 / readonly 63)**으로 정정되어, 요약·본문·코드 3자가 일치한다.
- 하단 "runtime 필드" 서술이 airflow.conf + **mock-api 3필드(responseTemplate·latencyMs·statusCode)** 를 모두 포함한다.
- (교차 참조) mock-api의 3개 runtime 라벨은 backend 미구현 상태임을 문서에 각주로 명기해 별도 계획서(격차 해소)와 연결한다.

## 접근 방법

### A. 누락 도구 2개를 "도구별 설정 API" 표에 추가 (문서 11~26행 영역)

1. **zookeeper 행**: 런타임 변경 API 없음(readonly 전용). 비고 = "Zookeeper 설정 — ui-backend 연동 없음; DAG/인프라 정의값". 엔드포인트 칸은 "–" 또는 "N/A(설정 파일)".
2. **mock-api 행**: `responseTemplate`/`latencyMs`/`statusCode`가 프론트에서 runtime 라벨이나 **backend 미구현**임을 명시. 비고 = "프론트 configField는 runtime 라벨이나 ui-backend RUNTIME_KEYS 미포함 — 실제 readonly 반려(격차 계획서 참조)". 지원 여부 칸 = ❌(라벨상 runtime, 실동작 미구현).

### B. applyMode 분류 하위 표 2개 추가 (문서 38~197행 영역, 마지막 kibana 표 뒤)

코드(`toolCatalog.ts`) 값 그대로 옮긴다:

- **zookeeper** (3): `connectString`(readonly, "Zookeeper 접속 문자열 — 인프라 정의값"), `sessionTimeout`(readonly, "세션 타임아웃 — 설정 파일/재기동"), `tickTime`(readonly, "tickTime — 설정 파일/재기동"). *근거 문구는 다른 도구 톤에 맞춰 작성; 실제 applyMode는 코드가 모두 readonly이므로 그대로 둔다.*
- **mock-api** (6): `endpoint`(readonly), `method`(readonly), `responseTemplate`(runtime†), `latencyMs`(runtime†), `statusCode`(runtime†), `authToken`(readonly). †표시 필드에 각주: "프론트 라벨 runtime이나 ui-backend 미구현 — 현재 readonly 반려".

### C. 분류 요약 통계 정정 (문서 201~212행)

전체 15개 도구 재집계 결과로 교체한다. 재계산 산출식(검증용):

| applyMode | 필드 수 | 비율 | 산출 |
|-----------|--------|------|------|
| runtime | 4 | 4.0% | airflow.conf(1) + mock-api(3) |
| restart | 27 | 26.7% | presidio1+docling6+kure5+valkey4+mysql6+es3+kibana2 |
| code | 7 | 6.9% | airflow(3: triggerRule/retries/retryDelay) + airflow-branch(4) |
| readonly | 63 | 62.4% | 나머지 |
| **합계** | **101** | 100% | 15개 도구 전 필드 |

- 현 요약(runtime 1 / restart 19 / code 7 / readonly 61 / 합계 88)은 **누락 도구 미반영 + 자체 산출 오류** 양쪽이므로 전량 교체한다.
- 각 도구 필드 수: nifi8·debezium9·dam6·airflow7·presidio6·docling6·kure6·valkey8·airflow-branch4·s3·mysql9·es9·kibana6·zookeeper3·mock-api6 = 101. (문서 작성 시 이 합을 한 번 더 검산한다.)

### D. 하단 runtime 필드 서술 갱신 (문서 211~213행)

- "runtime 필드" 목록에 `mock-api.responseTemplate`·`mock-api.latencyMs`·`mock-api.statusCode`를 추가하되, **"(프론트 라벨 기준; backend 미구현 — 격차 계획서 참조)"** 단서를 붙인다.
- `is_paused`·`variable.{key}` 미등록 비고는 유지하되, 별도 등록 계획서가 있음을 한 줄로 상호 참조한다.
