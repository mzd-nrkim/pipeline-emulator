# DAG 뷰 노드상세·실행이력 사이드바 → 모달 전환

> 작성일: 2026-07-20 / 상태: 초안
> 대상: `frontend/src/routes/[mode=mode]/pipeline/+page.svelte`

파이프라인 DAG 뷰에서 노드를 클릭하면 나오는 "노드 상세 / 실행 이력"이 현재 우측 고정 오버레이 드로어(사이드바, `w-80`)로 표시된다. 이를 뷰포트 중앙 모달(Dialog)로 전환한다. 사이드바는 폭(20rem)이 좁아 실행 이력의 비교(diff) 뷰가 답답하고, 캔버스를 가려 노드 배치 파악을 방해한다. 모달은 넓은 표시 공간과 배경 딤 처리로 두 문제를 해소한다.

---

## 목표

- 노드 클릭 또는 "실행 이력" 열기 시, 우측 드로어 대신 중앙 모달(Dialog)로 노드 상세·실행 이력이 표시된다.
- 모달은 노드 상세 / 실행 이력 두 탭을 그대로 유지하며, 기존 콘텐츠(설정 폼·DAG 트리거·실행 이력 diff)가 동일하게 동작한다.
- 모달은 배경 오버레이(딤)·ESC·바깥 클릭·닫기 버튼으로 닫히고, 접근성(포커스 트랩·`aria`)을 갖춘다.
- 실행 이력 비교 뷰가 사이드바 대비 넓은 폭에서 쾌적하게 보인다.

## 접근 방법

1. **컴포넌트 선택**: 이미 의존성에 포함된 `bits-ui`(v1.8.0)의 `Dialog`를 사용한다. 신규 라이브러리 도입 없이 `Dialog.Root / Trigger / Portal / Overlay / Content / Close` 프리미티브로 구성한다. 별도 wrapper 컴포넌트(`src/lib/components/ui/Dialog.svelte` 등 스타일 프리셋)를 만들지, `+page.svelte`에 인라인으로 둘지는 재사용 필요성 기준으로 결정한다(현재 사용처가 1곳이므로 인라인 우선, 추후 추출).

2. **상태 매핑**: 기존 상태 변수를 Dialog 제어로 연결한다.
   - `drawerOpen`(boolean) → `Dialog.Root`의 `open` 바인딩(`bind:open`). 변수명은 `dialogOpen`으로 정리하거나 기존명 유지.
   - `drawerTab`('node' | 'history') → 모달 헤더의 탭 전환 상태로 그대로 사용.
   - `selectedNode`, `triggeredRunId`, `triggerError`, 비교용 `selectedRunId`/`selectedRunIdB` → 변경 없음(콘텐츠 로직 재사용).
   - `handleNodeSelect`: 노드 선택 시 `dialogOpen = true` + `drawerTab = 'node'` 세팅(현재는 드로어가 상시 존재해 `drawerOpen`을 별도로 안 켰을 수 있음 — 열림 트리거 명시).

3. **마크업 이식**: 현재 드로어 블록(약 L160~522)의 헤더(탭+닫기)와 콘텐츠(노드 상세 탭 / 실행 이력 탭)를 `Dialog.Content` 내부로 옮긴다.
   - 컨테이너 클래스 교체: `absolute right-0 top-0 h-full w-80 ...` → 모달용 중앙 배치·최대폭(예: `max-w-3xl w-[90vw] max-h-[85vh]`)·`overflow-auto`. 실행 이력 diff를 고려해 노드 상세보다 넓은 폭 확보.
   - 닫기 버튼(`✕`)은 `Dialog.Close`로 대체. 노드 상세 탭 내부의 "노드 선택 해제(`selectedNode = null`)" 버튼은 모달을 닫지 않고 노드만 해제하는 기존 동작 유지(모달 닫힘과 구분).

4. **트리거 진입점 정리**: 현재 드로어를 여는 경로를 점검한다 — (a) 노드 클릭(`onnodeselect`→`handleNodeSelect`), (b) "실행 이력" 별도 열기 버튼이 있는지. 각 진입점이 `dialogOpen`을 켜도록 통일한다.

5. **접근성·UX**: bits-ui Dialog가 포커스 트랩·ESC·`aria-modal`·바깥 클릭 닫기를 기본 제공하므로 수동 `aria-label`만 유지·보완. 배경 `Dialog.Overlay`에 딤 스타일 적용. Portal 사용으로 z-index 이슈(기존 `z-20`) 해소.

6. **정리**: 전환 후 사용하지 않는 드로어 전용 클래스·주석("오버레이 드로어")·죽은 상태 변수를 제거한다.
